# Менеджер бэкендов

Менеджер бэкендов — это унифицированный диалог для управления всеми конфигурациями бэкендов. Через него вы можете добавлять, редактировать, удалять и проверять подключения к бэкендам.

## Как открыть

- **Меню**: **Инструменты → Менеджер бэкендов**

## Макет интерфейса

```
┌─────────────────────────────────────────────────┐
│  Менеджер бэкендов                      [Отмена] [Сохранить] │
├─────────────────────────────────────────────────┤
│  [ACP] [SkillRunner] [Generic HTTP]              │
├─────────────────────────────────────────────────┤
│  ACP                                   [Добавить ACP] │
│                                                 │
│  ┌─ Отображаемое имя: [________]  ─┐               │
│  │  Команда:      [________]    │               │
│  │  Аргументы:    Редактор аргументов │             │
│  │  Переменные окружения: Редактор переменных окружения  │  [Удалить]  │
│  └──────────────────────────────┘               │
│                                                 │
│  ┌─ Отображаемое имя: [________]  ─┐               │
│  │  ...                       │  [Удалить]      │
│  └──────────────────────────────┘               │
└─────────────────────────────────────────────────┘
```

## Общие операции

### Переключение вкладок

В верхней части диалога находятся три вкладки: **ACP**, **SkillRunner** и **Generic HTTP**. Нажмите на вкладку, чтобы переключиться в соответствующую область конфигурации типа бэкенда. Каждая вкладка содержит список всех настроенных бэкендов этого типа.

### Добавление бэкенда

Нажмите кнопку **Добавить** под вкладкой, чтобы создать новую пустую строку конфигурации для этого типа. Заполните поля и нажмите **Сохранить** в правом нижнем углу для применения.

### Редактирование бэкенда

Измените поля непосредственно в строке конфигурации. Несохранённые изменения не вступят в силу.

### Удаление бэкенда

Нажмите кнопку **Удалить** в строке конфигурации, чтобы удалить этот бэкенд. Удаления вступают в силу после сохранения.

### Сохранить и Отмена

| Кнопка | Расположение | Функция |
|--------|----------|----------|
| **Сохранить** | Правый нижний угол диалога | Сохранить все изменения и закрыть диалог |
| **Отмена** | Правый нижний угол диалога (рядом с Сохранить) | Отменить все несохранённые изменения и закрыть диалог |

Если перед закрытием диалога есть несохранённые изменения, появится запрос подтверждения.

## Вкладка ACP

Бэкенды ACP — это локально запущенные подпроцессы агентов. Конфигурация указывает команду запуска, а плагин управляет жизненным циклом процесса.

<figure class="zs-doc-figure"><img src="chrome://zotero-skills/content/help-docs/assets/img/docs/backends/backend-manager_ACP.webp" alt="Страница конфигурации бэкенда ACP" title="Страница конфигурации бэкенда ACP" loading="lazy" /><figcaption>Страница конфигурации бэкенда ACP</figcaption></figure>

### Описание полей

| Поле | Обязательно | Описание |
|-------|----------|-------------|
| **Отображаемое имя** | Да | Отображаемое имя для бэкенда, используется для идентификации в Панели мониторинга и боковой панели |
| **Команда** | Да | Команда для запуска бэкенда ACP (например, `npx opencode-ai@latest acp`) |
| **Аргументы** | Нет | Дополнительные аргументы для команды, добавляются по одному через редактор аргументов |
| **Переменные окружения** | Нет | Дополнительные переменные окружения, добавляются по одной через редактор переменных окружения (пары ключ-значение) |

### Пресеты ACP

В верхней части вкладки ACP есть выпадающий список **Добавить из пресета**. После выбора пресета плагин автоматически заполняет команду и общие параметры.

Встроенные пресеты:

| Пресет | Команда |
|--------|---------|
| **OpenCode** | `npx opencode-ai@latest acp` |
| **Codex** | `npx codex acp` |
| **Claude Code** | `npx @anthropic-ai/claude-code acp` |
| **Gemini CLI** | `npx @google/gemini-cli acp` |
| **Qwen Code** | `qwen-code acp` |

Вы всё ещё можете вручную изменить любое поле после выбора пресета.

### Кнопки действий

| Кнопка | Функция |
|--------|----------|
| **Обновить параметры выполнения** | Повторно определить список моделей, список режимов и другие возможности выполнения бэкенда |

### Редактор аргументов

**Добавить аргумент**: Нажмите кнопку добавления и введите содержимое аргумента.
**Удалить аргумент**: Нажмите кнопку удаления рядом с аргументом.

### Редактор переменных окружения

**Добавить переменную окружения**: Нажмите кнопку добавления и заполните Key и Value.
**Удалить переменную окружения**: Нажмите кнопку удаления рядом с переменной.

## Вкладка SkillRunner

Бэкенды SkillRunner взаимодействуют с сервисами Skill-Runner через HTTP API, поддерживая как локальные, так и удалённые режимы развёртывания.

<figure class="zs-doc-figure"><img src="chrome://zotero-skills/content/help-docs/assets/img/docs/backends/backend-manager_skillrunner.webp" alt="Страница конфигурации бэкенда SkillRunner" title="Страница конфигурации бэкенда SkillRunner" loading="lazy" /><figcaption>Страница конфигурации бэкенда SkillRunner</figcaption></figure>

### Описание полей

| Поле | Обязательно | Описание |
|-------|----------|-------------|
| **Отображаемое имя** | Да | Отображаемое имя для бэкенда |
| **Base URL** | Да | Адрес сервиса Skill-Runner (например, `http://127.0.0.1:29813`) |
| **Аутентификация** | Нет | Выберите `none` (без аутентификации) или `bearer` (аутентификация Bearer Token) |
| **Токен авторизации** | Нет | Bearer Token (заполняется только когда аутентификация установлена в bearer) |
| **Тайм-аут** | Нет | Тайм-аут запроса (миллисекунды) |

### Кнопки действий

| Кнопка | Функция |
|--------|----------|
| **Открыть UI управления** | Открыть встроенный веб-интерфейс управления Skill-Runner |
| **Обновить кэш моделей** | Обновить кэш списка моделей для этого бэкенда |

## Вкладка Generic HTTP

Бэкенды Generic HTTP используются для отправки запросов на любой HTTP-сервис, в основном для вызова внешних API (таких как сервис разбора документов MinerU).

<figure class="zs-doc-figure"><img src="chrome://zotero-skills/content/help-docs/assets/img/docs/backends/backend-manager_generic-HTTP.webp" alt="Страница конфигурации бэкенда Generic HTTP" title="Страница конфигурации бэкенда Generic HTTP" loading="lazy" /><figcaption>Страница конфигурации бэкенда Generic HTTP</figcaption></figure>

### Описание полей

| Поле | Обязательно | Описание |
|-------|----------|-------------|
| **Отображаемое имя** | Да | Отображаемое имя для бэкенда |
| **Base URL** | Да | Базовый адрес HTTP-сервиса |
| **Аутентификация** | Нет | Выберите `none` или `bearer` |
| **Токен авторизации** | Нет | Bearer Token (заполняется только когда аутентификация установлена в bearer) |
| **Тайм-аут** | Нет | Тайм-аут запроса (миллисекунды) |

## Обнаружение возможностей бэкенда

После сохранения бэкенда плагин автоматически обнаруживает возможности бэкенда в фоновом режиме:

- **ACP**: Проверяет доступность команды, инициализацию подключения, список моделей, список режимов и вычисляет отпечаток конфигурации для обнаружения последующих изменений
- **SkillRunner**: Проверяет доступность API, список движков, список моделей
- **Generic HTTP**: Проверяет доступность конечной точки HTTP

Результаты обнаружения отображаются как индикаторы статуса бэкенда в Панели мониторинга и боковой панели.
