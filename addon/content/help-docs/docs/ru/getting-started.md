# Начало работы

## 1. Установите официальные пакеты Workflow

Сам плагин не содержит бизнес-логики. После установки плагина вам нужно сначала установить официальные пакеты Workflow:

1. Щёлкните правой кнопкой мыши по любому элементу Zotero → **Zotero Agents** → **📦 Установить официальные пакеты Workflow**
2. Дождитесь завершения загрузки и установки
3. После успешной установки все официальные Workflow будут видны в Панели мониторинга

Вы также можете в любое время установить или обновить официальные пакеты через **Zotero → Настройки → Zotero Agents**.

## 2. Настройте бэкенд

### Бэкенд ACP (рекомендуется)

Это наиболее рекомендуемый способ — если у вас установлен любой совместимый с ACP инструмент агента на вашем компьютере, он не требует никакой дополнительной настройки.

1. Откройте **Инструменты → [Менеджер бэкендов](#doc/backends%2Fbackend-manager)**
2. Переключитесь на вкладку **ACP**
3. Выберите ваш инструмент агента из выпадающего списка **Добавить из пресета** (Codex / OpenCode / Claude Code и т.д.)
4. Пресет автоматически заполняет команду; нажмите **Сохранить** в правом нижнем углу

**Впервые используете инструмент агента?** Обратитесь к официальной документации соответствующего инструмента для установки:

| Агент | Руководство по установке |
|-------|-------------------|
| **OpenCode** | [Документация opencode.ai](https://opencode.ai/docs) |
| **Codex** | [Документация OpenAI Codex](https://platform.openai.com/docs) |
| **Claude Code** | [Документация Anthropic](https://docs.anthropic.com/en/docs/claude-code) |
| **Gemini CLI** | [Документация Google](https://github.com/google-gemini/gemini-cli) |
| **Qwen Code** | [Документация Alibaba Cloud](https://help.aliyun.com/zh/model-studio/qwen-code) |

→ Подробнее см. [Конфигурация бэкенда ACP](#doc/backends%2Facp)

### Бэкенд MinerU (для разбора PDF)

Workflow MinerU может конвертировать PDF в Markdown, что делает его идеальным этапом предварительной обработки для всего последующего анализа литературы. Настройка проста:

1. Посетите [mineru.net](https://mineru.net) для регистрации учётной записи и получите API-токен в разделе **API → Управление API**
2. Откройте **Инструменты → [Менеджер бэкендов](#doc/backends%2Fbackend-manager)**
3. Переключитесь на вкладку **Generic HTTP**, нажмите **Добавить Generic HTTP**
4. Заполните: Отображаемое имя `MinerU Official` · Base URL `https://mineru.net` · Аутентификация `bearer` · Токен авторизации: вставьте ваш API-токен · Тайм-аут `600000`
5. Нажмите **Сохранить** в правом нижнем углу

→ Подробнее см. [Руководство по использованию MinerU](#doc/workflows%2Fmineru)

### Альтернатива: Skill-Runner, развёрнутый в Docker

Если вам нужно постоянное фоновое выполнение или общий доступ в локальной сети, вы можете [развернуть Skill-Runner с помощью Docker](#doc/backends%2Fskill-runner#рекомендуется-постоянное-развёртывание-в-docker). После развёртывания добавьте экземпляр бэкенда на вкладке SkillRunner.

> Подробные инструкции по эксплуатации см. в [Менеджере бэкендов](#doc/backends%2Fbackend-manager).

## 3. Полный Workflow

Ниже описан полный сквозной Workflow. Рекомендуется попробовать каждый шаг по порядку. Сначала выберите статью с PDF-вложением из вашей библиотеки.

### Шаг 1: PDF → Markdown (MinerU)

Щёлкните правой кнопкой мыши по этой статье (или напрямую по её PDF-вложению) и выберите **Zotero Agents → MinerU**. Через короткое время в том же каталоге, что и PDF, будет создан файл `.md` с содержанием статьи.

### Шаг 2: Попробуйте встроенный Markdown-ридер

Найдите только что созданный файл `.md` в списке вложений Zotero и **дважды щёлкните, чтобы открыть его во встроенном ридере** — с навигацией по оглавлению, поиском, рендерингом математических формул и подсветкой синтаксиса кода. Если вы не хотите использовать встроенный ридер, вы можете отключить его в Настройках и вернуться к системному открывателю по умолчанию.

→ Подробнее см. [Встроенный Markdown-ридер](#doc/markdown-reader)

### Шаг 3: Запустите анализ литературы

Щёлкните правой кнопкой мыши по этой статье (или напрямую по вложению `.md`) и выберите **Zotero Agents → Анализ литературы**. Агент автоматически сгенерирует три артефакта; после завершения под элементом появятся три вложения-заметки:

| Заметка | Содержание |
|------|---------|
| **Дайджест** | Дайджест статьи — исследовательский контекст, методы, результаты и выводы |
| **Ссылки** | Структурированные ссылки — табличный список цитирований |
| **Анализ цитирований** | Отчёт по анализу цитирований — контекст цитирования и классификация намерений цитирования |

→ Подробнее см. [Анализ литературы](#doc/workflows%2Fliterature-analysis)

### Шаг 4: Интерактивный пояснитель литературы

Если у вас есть вопросы по этой статье, щёлкните правой кнопкой мыши и выберите **Zotero Agents → Пояснитель литературы**. Боковая панель автоматически откроет панель чата, где вы можете свободно общаться с агентом о содержании статьи. Ответы агента проходят через шлюз верификации, так что вам не нужно беспокоиться о вымысле. После разговора запись вопросов и ответов будет сгенерирована в виде конспекта.

→ Подробнее см. [Пояснитель литературы](#doc/workflows%2Fliterature-explainer)

### Шаг 5: Глубокое чтение

Когда вам нужно тщательно и системно прочитать важную статью, щёлкните правой кнопкой мыши и выберите **Zotero Agents → Глубокое чтение**. Агент создаст оформленный автономный HTML-документ — с анализом разделов, ключевыми концепциями, ссылками и двуязычными переводами. Обогащённый информацией вашей библиотеки (если она доступна), этот документ также будет содержать более широкий исследовательский контекст, связанные концепции и ключевые вопросы.

→ Подробнее см. [Глубокое чтение](#doc/workflows%2Fliterature-deep-reading)

### Шаг 6: Синтез тем — от отдельных статей к общей картине

Когда ваша библиотека достигла определённого размера и соответствующие статьи прошли анализ литературы и нормализацию тегов, вы можете создать Topic Synthesis.

Запустите **Создать Topic Synthesis** в Панели мониторинга, введите описание вашего исследовательского направления, и агент автоматически определит соответствующие статьи в вашей библиотеке и сгенерирует крайне строгий, точный и всеобъемлющий отчёт синтеза. Этот отчёт написан исключительно на основе содержимого вашей библиотеки, что гораздо точнее и надёжнее, чем общие ответы ИИ.

→ Подробнее см. [Синтез тем](#doc/workflows%2Ftopic-synthesis)

## Следующие шаги

- **Пакетная обработка**: Запустите [Анализ литературы](#doc/workflows%2Fliterature-analysis) пакетно для статей в вашей библиотеке, чтобы создать основу для Synthesis
- **Система тегов**: Используйте [Tag Bootstrapper](#doc/workflows%2Ftag-bootstrapper) для создания контролируемого словаря и стандартизации ваших метаданных
- **Исследование графа**: Визуализируйте вашу сеть цитирований в [Synthesis Workbench](#doc/synthesis%2Findex)
- **Собственная разработка**: Обратитесь к разделу [Пользовательские Workflow](#doc/workflows%2Fcustom%2Findex) для создания собственных Workflow
- **Сообщение о проблемах**: Сообщайте о проблемах на [GitHub](https://github.com/leike0813/zotero-agents/issues) или [Gitee](https://gitee.com/leike0813/zotero-agents/issues)
