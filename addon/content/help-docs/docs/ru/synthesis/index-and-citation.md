# Индекс и граф цитирований

## Поверхность индекса

На странице Synthesis Workbench → Индекс вы можете управлять индексом канонических ссылок. Поверхность индекса содержит **два подпредставления**:

### Представление реестра

Отображает список всех отслеживаемых статей в библиотеке, где каждая строка показывает статью и её статус покрытия:

- **Информация о статье**: Название, автор, год
- **Покрытие**: Complete / Partial / Missing (статус покрытия трёх типов артефактов: резюме, ссылки, анализ цитирований)
- **Развернуть строку**: При развёртывании показывает список ссылок статьи, где каждое цитирование отмечено статусом привязки (unbound / candidate / accepted / rejected)
- **Фильтрация**: Фильтр по области (all / library / cited), покрытию или поиск

### Представление канонических ссылок

Отображается, когда активный инструмент индексирования переключён на «Revise Canonical»:

- **Список канонических ссылок**: Дедуплицированные канонические записи ссылок
- **Поиск и фильтрация**: Фильтр по статусу привязки, видимости в графе, статусу перенаправления или наличию дубликатов-кандидатов
- **Действия**: Редактирование метаданных, объединение дубликатов ссылок, создание перенаправлений, просмотр элементов рецензирования

## Индекс канонических ссылок

Индекс канонических ссылок — это основной индекс системы Synthesis, выполняющий дедупликацию и каноникализацию всех ссылок из статей в библиотеке. Он получает исходные данные о цитированиях из Reference Sidecar (см. раздел «Reference Sidecar» в [обзоре](#doc/synthesis%2Findex)) и формирует индекс через извлечение, каноникализацию и привязку совпадений.

### Возможности

- **Полнотекстовый поиск**: Поиск по всем каноникализированным ссылкам
- **Редактирование метаданных**: Изменение метаданных записей цитирований
- **Объединение**: Объединение дублирующихся записей ссылок (автоматически создаёт перенаправления)
- **Перенаправление**: Направление одной ссылки на другую каноническую запись
- **Рецензирование**: Просмотр элементов рецензирования качества привязки цитирований
- **Дедупликация**: Обнаружение потенциальных дубликатов ссылок

### Типы записей ссылок

| Тип | Описание |
|------|-------------|
| **Bound** | Связана с элементом в библиотеке Zotero |
| **External** | Известная литература, не входящая в текущую библиотеку Zotero |
| **Unresolved** | Извлечена из ссылок, но ещё не идентифицирована |

## Конвейер сопоставления ссылок

Сопоставление ссылок — это процесс автоматического установления ассоциаций между ссылками, извлечёнными из статей, и элементами в библиотеке Zotero. Система использует **двухэтапную модель** для баланса производительности и точности.

### Двухэтапная модель

#### Этап 1: Лёгкое обновление Sidecar

Запускается во время обычных операций (например, после применения резюме), сканирует статус Sidecar, сравнивает хеши артефактов цитирований и обрабатывает только изменившиеся ссылки. **Не выполняет продвинутую дедупликацию или построение индекса**, выполняет только лёгкую каноническую привязку и назначение.

- Триггер: После завершения выполнения Workflow и записи артефактов или через явную операцию обновления
- Область: Инкрементная (только изменившиеся ссылки)
- Алгоритм: Простое сопоставление идентификаторов (DOI, arXiv, ISBN)

#### Этап 2: Продвинутое сопоставление цитирований

Явно запускаемая операция глубокого сопоставления. Строит полный индекс совпадений цитирований и запускает комплексные алгоритмы сопоставления и дедупликации.

- Триггер: Ручной запуск пользователем, периодическое обслуживание
- Область: Полная
- Алгоритм: Многостратегийное сопоставление + кластерная дедупликация

:::caution Примечание о производительности
Продвинутое сопоставление цитирований, обновление индекса и перестроение графа цитирований требуют значительных вычислительных ресурсов. Поскольку Zotero использует архитектуру с единым хост-процессом, эти операции могут вызывать кратковременные задержки интерфейса во время выполнения. Пожалуйста, проявите терпение. Решение этой проблемы запланировано в будущем рефакторинге архитектуры.
:::

### Стратегии сопоставления

| Стратегия | Основа совпадения | Уверенность | Описание |
|----------|-------------|------------|-------------|
| Сопоставление DOI | Идентификатор DOI | Детерминированная | Точное совпадение, автопринятие |
| Сопоставление arXiv | Идентификатор arXiv | Детерминированная | Точное совпадение, автопринятие |
| Сопоставление ISBN | Номер ISBN | Детерминированная | Точное совпадение, автопринятие |
| Сходство названий | Нечёткое совпадение названий | Высокая / средняя / низкая | Использует стандартизированные названия и компактные названия для расчёта сходства |
| Автор + год | Имена авторов и год публикации | Средняя / низкая | Комбинирует нормализацию авторов и диапазон лет для сопоставления |

### Уровни уверенности

| Уровень | Описание | Рекомендуемое действие |
|-------|-------------|-------------------|
| `deterministic` | Детерминированное совпадение | Автопринятие |
| `high` | Высокая уверенность | Приемлемо |
| `medium` | Средняя уверенность | Рекомендуется рецензирование |
| `low` | Низкая уверенность | Требуется рецензирование |
| `review` | Требуется суждение человека | Обязательное рецензирование |

### Кластерная дедупликация

Этап продвинутого сопоставления выполняет кластерную дедупликацию канонических ссылок. Процесс алгоритма:

1. Построение записи дедупликации для каждой канонической ссылки (включая фильтрацию допустимости и анализ библиографического шума)
2. Попарное сравнение генерирует рёбра кластеров (точное совпадение идентификаторов, каноническое совпадение названий, нечёткое совпадение названий и т. д.)
3. Рёбра агрегируются в кластеры и подкластеры
4. Генерируются автоматические перенаправления или предложения рецензирования для дедупликации

Ограничение безопасности: Совпадения с низкой уверенностью (например, `contained_extension_risk`) никогда не запускают автоматические перенаправления и требуют рецензирования пользователем.

### Поверхность рецензирования

В [центре рецензирования](#doc/synthesis%2Freview) вы можете просматривать и обрабатывать предложения совпадений цитирований, принимая или отклоняя их по одному.

## Граф цитирований

Граф цитирований визуализирует статьи в библиотеке и их ссылки в виде сетевого графа. Данные графа строятся как проекция SQLite и могут допускать определённую степень устаревания данных (не является зеркалом реального времени).

<figure class="zs-doc-figure"><img src="chrome://zotero-skills/content/help-docs/assets/img/docs/synthesis/citation-graph.webp" alt="Synthesis Citation Graph" title="Synthesis Citation Graph" loading="lazy" /><figcaption>Synthesis Citation Graph</figcaption></figure>

### Типы узлов

| Узел | Цвет | Описание |
|------|-------|-------------|
| `library_paper` | Синий | Статьи, уже находящиеся в библиотеке Zotero |
| `external_reference` | Зелёный | Известные ссылки, не входящие в библиотеку |
| `unresolved_reference` | Серый | Извлечённые, но неидентифицированные ссылки |

### Информация о рёбрах

Каждое ребро цитирования содержит:

- **mention_count**: Количество упоминаний
- **primary_role**: Основная роль цитирования (например, background, comparison, support, contrast)
- **aux_roles**: Список вспомогательных ролей
- **role_evidence**: Обоснование определения роли

### Метрики графа

Граф цитирований может вычислять различные метрики для выявления ключевых статей и влиятельных работ:

| Метрика | Описание |
|--------|-------------|
| **Количество цитирований** | Общее количество цитирований статьи |
| **PageRank** | Оценка важности узла на основе структуры графа |
| **Foundation Score** | Степень, в которой работа является фундаментальной в области |
| **Frontier Score** | Степень, в которой работа представляет передовые исследования |

### Раскладки визуализации

| Раскладка | Описание | Вариант использования |
|--------|-------------|----------|
| **Force (Force-Directed)** | Раскладка d3-force | Исследование общей структуры |
| **Radial** | Развёртывание вокруг выбранного узла | Анализ сети цитирований статьи |
| **Components** | Группировка по связным компонентам | Обнаружение независимых кластеров цитирований |

### Интерактивные операции

- **Масштабирование / перемещение**: Свободный просмотр графа
- **Наведение**: Просмотр меток узлов и основной информации
- **Клик по узлу**: Открытие соответствующего элемента статьи в Zotero
- **Фильтрация**: Фильтрация отображаемых цитирований по роли, теме или типу узла
- **Переключение слабых цитирований**: Показ/скрытие рёбер с низким количеством цитирований
- **Ползунок глубины**: Управление глубиной развёртывания сети цитирований

### Фильтрация по темам

Вы можете фильтровать граф цитирований по темам, чтобы показывать только статьи и связи цитирования, относящиеся к определённым темам. Области тем отображаются в графе различными цветами и группами.
