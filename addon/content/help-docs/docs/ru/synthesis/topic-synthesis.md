# Создание синтеза темы

## Что такое синтез темы?

Синтез темы — это процесс систематического анализа и обобщения группы связанной литературы. Он автоматически извлекает ключевую информацию, определяет структуру темы и генерирует комплексные аналитические отчёты через AI-воркфлоу.

## Поверхность тем

На странице Synthesis Workbench → Темы вы можете просматривать и управлять всеми созданными темами. Поверхность тем поддерживает **три режима просмотра**:

| Вид | Описание | Вариант использования |
|------|-------------|----------|
| **Граф** | Force-directed граф с темами в качестве узлов и связями в качестве рёбер | Интуитивное понимание межтемных ассоциаций |
| **Сетка** | Карточки с названием, количеством статей, резюме и кнопками действий | Просмотр и поиск тем |
| **Список** | Табличный вид с колонками: название, количество статей, дата создания, дата обновления, статус | Сортировка и пакетные операции |

### Операции управления темами

- **Поиск**: Поиск по названию и описанию темы
- **Сортировка**: Сортировка по названию, количеству статей или дате обновления
- **Создать новую тему**: Нажмите кнопку создания для запуска конвейера Workflow
- **Обновить тему**: Повторный запуск конвейера для обновления анализа темы
- **Удалить тему**: Удаление больше не нужных тем

## Процесс создания

Создание темы управляется Workflow и представляет собой многошаговый автоматизированный конвейер:

```
1. create-topic-prepare
   → Сбор данных литературы, формирование набора статей
   
2. topic-synthesis-core-enrichment
   → Основное обогащение: извлечение информации, ассоциация знаний
   
3. topic-synthesis-finalize
   → Генерация итоговых артефактов анализа и отчётов

(update-topic-synthesis-prepare используется для обновления существующих тем)
```

### Предварительные требования

- Настроенный [бэкенд Skill-Runner](#doc/backends%2Fskill-runner)
- Релевантные статьи в библиотеке
- Статьи с сгенерированными резюме и анализом цитирований (опционально, рекомендуется)

Этот конвейер управляется Workflow [Создание синтеза темы](#doc/workflows%2Ftopic-synthesis).

## Инспектор тем

После создания темы нажмите на неё, чтобы войти в инспектор тем. Это многостраничный ридер, содержащий 8 вкладок, каждая из которых представляет измерение темы.

### Обзор

- Название темы, описание, оценка важности
- Сводка ключевых утверждений
- Статистика (количество статей, количество категорий, количество утверждений и т. д.)
- Информация о расположении в графе тем

### Таксономия

Отображает иерархическую структуру классификации темы:

- Более широкие темы: Области более широких тем
- Более узкие темы: Более конкретные подтемы
- Связанные темы: Другие ассоциированные темы
- Положение и иерархия в графе тем

### Утверждения

Ключевые утверждения и положения, извлечённые из литературы:

- Каждое утверждение включает исходные цитирования-доказательства
- Указаны статьи, из которых взяты утверждения
- Тип утверждения (находки / гипотезы / выводы и т. д.)
- Количество статей, поддерживающих утверждение

### Сравнение

Сравнение точек зрения различных статей по одной теме:

- Измерения сравнения (методы, выводы, наборы данных и т. д.)
- Позиция и аргументы каждой статьи
- Визуализация консенсуса и расхождений

### Будущие направления

Пробелы в исследованиях и будущие направления, выявленные в результате анализа литературы:

- Открытые вопросы
- Потенциальные направления исследований
- Связанные задачи и рекомендации

### Покрытие

Анализирует степень охвата темой соответствующей литературы:

- Список статей, охваченных темой
- Полнота статей (наличие артефактов резюме/анализа цитирований)
- Охваченные и неохваченные аспекты

### Ссылки

Все ссылки, ассоциированные с темой, включая детали привязки:

- Ссылка на элемент Zotero для каждого цитирования
- Роль цитирования в теме (support / contrast / background)
- Источник и контекст цитирования

### Отчёт (полный отчёт)

Сгенерированный структурированный аналитический отчёт синтеза (в формате Markdown):

- Полный текст анализа темы
- Может быть экспортирован в Markdown или автономный HTML
- Подходит для использования в качестве справочного материала в академическом письме

## Граф тем

Граф тем — это иерархическая сеть тем, показывающая связи между темами:

### Типы узлов

| Тип | Описание |
|------|-------------|
| **materialized** | Структурированные темы, которые были фактически созданы |
| **placeholder** | Заготовки тем, существование которых выведено, но ещё не созданы |

### Статус рёбер

| Статус | Описание |
|--------|-------------|
| `suggested` | Предложенные системой связи (ожидают рецензирования) |
| `confirmed` | Подтверждённые пользователем связи |
| `rejected` | Отклонённые пользователем связи |
| `stale` | Устаревшие данные, ожидающие переоценки |
| `deleted` | Удалённые связи |

### Типы связей

| Связь | Описание |
|--------------|-------------|
| `broader_than` | A — более широкая тема, чем B |
| `related_to` | Две темы связаны |
| `overlaps_with` | Две темы пересекаются |
| `contrasts_with` | Две темы противопоставлены друг другу |

### Управление темами

- **Создать новую тему**: Нажмите «Создать» на странице тем
- **Редактировать тему**: Изменить название, описание, важность и т. д.
- **Ассоциировать статьи**: Добавить или удалить статьи из темы
- **Просмотр графа тем**: Просмотр сети связей между темами
