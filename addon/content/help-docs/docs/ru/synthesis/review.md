# Центр рецензирования

Поверхность рецензирования — это централизованное место для обработки всех элементов, ожидающих рецензирования в системе Synthesis. Она содержит три вкладки: **Совпадения цитирований**, **Концепции** и **Граф тем**.

## Проверка совпадений цитирований

Когда система автоматически сопоставляет ссылки с элементами Zotero, совпадения, которые нельзя определить с уверенностью, направляются в очередь рецензирования как предложения.

### Статус предложения совпадения

| Статус | Описание |
|--------|-------------|
| **Ожидает** | Созданное системой совпадение, ожидающее подтверждения или отклонения пользователем |
| **Принято** | Пользователь подтвердил привязку; ссылка теперь связана с элементом Zotero |
| **Отклонено** | Пользователь отклонил привязку |
| **Возобновлено** | Ранее обработанное предложение, возобновлённое для повторного рецензирования |

### Доступные действия

- **Принять**: Подтвердить связь цитирования с элементом
- **Отклонить**: Отклонить предложение совпадения
- **Пакетные операции**: Выбрать несколько предложений для массового принятия или отклонения

### Уверенность совпадения

См. [Индекс и граф цитирований](#doc/synthesis%2Findex-and-citation) для описания уровней уверенности. Детерминированные совпадения и совпадения с высокой уверенностью обычно обрабатываются автоматически; совпадения со средней и более низкой уверенностью поступают в очередь рецензирования.

### Фильтрация и сортировка

Вы можете фильтровать список предложений по:

- Статусу совпадения (ожидает / принято / отклонено)
- Стратегии совпадения (DOI / название / автор и т. д.)
- Уровню уверенности
- Сортировка по времени или релевантности

## Рецензирование концепций

Автоматическое расширение базы знаний концепций может создавать предложения совпадений концепций с низкой уверенностью, требующие рецензирования и подтверждения пользователем.

### Объекты рецензирования

- **Предложения новых концепций**: Новые кандидаты концепций, автоматически извлечённые из литературы
- **Подтверждение смысла**: Подтверждение, когда к существующей концепции добавляется новое значение (смысл)
- **Предложения псевдонимов**: Подтверждение, когда обнаружено альтернативное название для одной концепции

### Как работать

Каждое предложение отображает название концепции, источник извлечения, уровень уверенности и подтверждающие данные. Вы можете:

- **Принять**: Подтвердить предложение и записать его в базу знаний концепций
- **Отклонить**: Отклонить предложение
- **Просмотреть контекст**: Посмотреть, где концепция встречается в литературе

## Рецензирование графа тем

Когда система обнаруживает потенциальные связи между темами, она генерирует предложения связей для рецензирования.

### Типы связей

| Связь | Описание |
|--------------|-------------|
| `broader_than` | A — более широкая тема, чем B |
| `related_to` | Две темы связаны |
| `overlaps_with` | Две темы имеют пересечение по содержанию |
| `contrasts_with` | Две темы противопоставлены друг другу |

### Содержание предложения

Каждое предложение отображает:

- **Исходная и целевая тема** — названия и описания
- **Предлагаемый тип связи**
- **Уверенность** (на основе семантического анализа содержания тем)
- **Подтверждающие данные** (совместно охваченные статьи и т. д.)

### Как работать

- **Принять**: Подтвердить связь и записать её в граф тем
- **Отклонить**: Отклонить предложение связи
- **Возобновить**: Повторно открыть ранее обработанное предложение для рецензирования
