# Обзор Synthesis Workbench

Synthesis Workbench — это платформа для глубокого анализа литературы, предоставляемая Zotero Agents. Она превращает вашу библиотеку в структурированную сеть знаний, поддерживая синтез тем, анализ цитирований, управление концепциями и контролируемый словарь тегов.

<figure class="zs-doc-figure"><img src="chrome://zotero-skills/content/help-docs/assets/img/docs/synthesis/home.webp" alt="Synthesis Workbench Home" title="Synthesis Workbench Home" loading="lazy" /><figcaption>Synthesis Workbench Home</figcaption></figure>

## Как открыть

1. Откройте Dashboard / Synthesis Workspace через **кнопку на панели инструментов** или **меню**
2. Переключитесь на вкладку **Synthesis** в Workspace Tab

## Все поверхности (страницы)

Synthesis Workbench состоит из 8 поверхностей, каждая из которых предоставляет функциональный вид:

| Поверхность | Функция | Документация |
|---------|----------|------|
| **Home** | Панель обзора библиотеки: статистика библиотеки (зарегистрированные статьи / количество тем / узлы графа), панель состояния Git-синхронизации, карточки актуальных тем | [Подробнее](#doc/synthesis%2Fhome) |
| **Topics** | Список и управление темами: 3 режима просмотра (граф / сетка / список), создание и обновление тем, поиск и сортировка тем | [Подробнее](#doc/synthesis%2Ftopic-synthesis) |
| **Index** | Индекс канонических ссылок: реестр статей (список статей + строки цитирований + статус привязки), обзор канонических ссылок (поиск / объединение / перенаправление / дедупликация) | [Подробнее](#doc/synthesis%2Findex-and-citation) |
| **Review** | Центр рецензирования: 3 вкладки — проверка совпадений цитирований (принять/отклонить предложения привязки), рецензирование концепций, рецензирование связей графа тем | [Подробнее](#doc/synthesis%2Freview) |
| **Graph** | Визуализация графа цитирований (force-directed / радиальная / по компонентам — 3 раскладки), с фильтрацией по темам и взаимодействием с узлами/рёбрами | [Подробнее](#doc/synthesis%2Findex-and-citation) |
| **Tags** | Управление контролируемым словарём тегов + одобрение предложений автоматической разметки | [Подробнее](#doc/synthesis%2Ftags) |
| **Concepts** | Управление базой знаний концепций: четырёхслойная структура концепции / смыслы / псевдонимы / отношения, с возможностью наложения на граф тем и ридер | [Подробнее](#doc/synthesis%2Fconcepts) |
| **Reader** | Ридер тем: полная страница деталей темы с 8 вкладками (Обзор, Таксономия, Утверждения, Сравнение, Будущие направления, Покрытие, Ссылки, Отчёт) | [Подробнее](#doc/synthesis%2Ftopic-synthesis) |

## Ключевые концепции

### Canonical Store

Canonical Store — это хранилище графа знаний, лежащее в основе системы Synthesis. Оно хранит JSON-файлы с контент-адресацией в каталоге данных Zotero.

**Расположение хранилища:** `<каталог данных Zotero>/zotero-agents/data/synthesis/`

**Структура каталогов:**

```
synthesis/
├── topics/             # Структурированные артефакты для синтеза тем
├── concepts/           # База знаний концепций
├── topic-graph/        # Узлы и рёбра графа тем
├── citation-graph/     # Снимки графа цитирований
├── tags/               # Контролируемый словарь тегов
├── sync/               # Рабочее дерево Git-синхронизации
└── state/              # Состояние среды выполнения (транзакции, квитанции, кэши и т. д.)
```

Каждый файл использует формат JSON-конверта (CanonicalEnvelope), который включает идентификатор схемы, номер версии, временную метку и тело данных, проверенное по схеме. Операции записи используют транзакционную семантику: данные сначала подготавливаются в каталоге транзакций, переносятся в каноническое расположение после успешной проверки и автоматически откатываются при сбое.

### Reference Sidecar

Reference Sidecar — это индекс прикреплённых артефактов для каждой статьи. Когда Workflow обрабатывает элемент литературы и генерирует резюме, список ссылок и анализ цитирований, эти артефакты прикрепляются к элементу в виде структурированных заметок (Zotero Notes). Система Sidecar сканирует эти заметки и записывает статус артефактов (полный / частичный / отсутствует) в индекс.

**Цикл сканирования Sidecar:** Sidecar запускает сканирование в следующие моменты:

- После завершения выполнения Workflow и записи артефактов
- При явном запуске операции обновления Sidecar
- При обнаружении системой устаревших данных Sidecar при запуске

**Типы артефактов:**

| Артефакт | Описание |
|----------|-------------|
| `digest` | Резюме статьи (Markdown) |
| `references` | Список ссылок (JSON) |
| `citation_analysis` | Отчёт анализа цитирований (JSON) |

Данные Sidecar служат основным входным источником для индекса канонических ссылок — система извлекает записи о цитированиях из артефакта ссылок, устанавливает канонические ссылки, а затем пытается сопоставить и привязать их к элементам библиотеки.

### Поток данных

```
Библиотека Zotero
    │
    ├──→ Выполнение Workflow (анализ литературы / глубокое чтение)
    │         │
    │         ↓
    │   Заметки артефактов (резюме / ссылки / анализ цитирований)
    │         │
    │         ↓
    │   Reference Sidecar ← Сканирование статуса артефактов
    │         │
    │         ├──→ Индекс канонических ссылок
    │         │         │
    │         │         ├──→ Привязка цитирований (привязка к элементам Zotero)
    │         │         └──→ Граф цитирований
    │         │
    │         └──→ Синтез тем
    │                   │
    │                   ├──→ Граф тем (связи между темами)
    │                   └──→ Ассоциации концепций (база знаний концепций)
    │
    └──→ Git Sync ←→ Удалённый репозиторий (контроль версий и резервное копирование)
```

## Предварительные требования

Для использования Synthesis Workbench необходимо:

- Настроенный бэкенд [Skill-Runner](#doc/backends%2Fskill-runner) (для запуска Workflow синтеза)
- Статьи уже присутствующие в библиотеке
