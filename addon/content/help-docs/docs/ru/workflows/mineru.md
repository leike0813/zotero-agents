# Разбор PDF MinerU

## Назначение

Вызов сервиса MinerU для разбора PDF-документов, извлечения высококачественного текста Markdown и изображений, создания непосредственно читаемых файлов заметок.

MinerU — это инструмент разбора PDF на основе глубокого обучения, который извлекает высококачественный текст и фигуры из академических статей.

## Сценарии использования

- Конвертация литературы в формате PDF в редактируемый Markdown
- Подготовка документов с простым текстом для нижестоящих Workflow (например, Анализ литературы, Глубокое чтение)
- Извлечение изображений и таблиц из PDF

## Настройка бэкенда MinerU

### 1. Регистрация учётной записи MinerU и получение API-токена

1. Посетите [mineru.net](https://mineru.net) для регистрации учётной записи
2. После входа перейдите на страницу **API → Управление API**
3. Создайте или скопируйте API-токен

### 2. Добавление бэкенда в Менеджере бэкендов

1. Откройте **Инструменты → [Менеджер бэкендов](#doc/backends%2Fbackend-manager)**
2. Переключитесь на вкладку **Generic HTTP**
3. Нажмите **Добавить Generic HTTP**
4. Заполните следующие поля:

| Поле | Значение |
|------|-----|
| Отображаемое имя | `MinerU Official` (или любое имя, которое вы предпочитаете) |
| Base URL | `https://mineru.net` |
| Метод аутентификации | `bearer` |
| Токен авторизации | Вставьте API-токен, полученный на предыдущем шаге |
| Тайм-аут | `60000` (60 секунд) |

5. Нажмите **Сохранить** в правом нижнем углу

## Ограничения ввода

| Тип ограничения | Описание |
|---------|------|
| Единица ввода | Вложение |
| Принимаемые типы | `application/pdf` (только PDF) |
| Обнаружение конфликтов | Если файл `.md` с тем же именем уже существует в том же каталоге, PDF пропускается |

### Способы запуска

- Прямой выбор одного или нескольких вложений PDF
- Выбор родительского элемента, и плагин автоматически раскроет его дочерние вложения PDF

### Обработка конфликтов

- Проверяет, существует ли `<имя файла PDF>.md` в целевом каталоге
- Если существует, ввод пропускается во время предварительной обработки
- Если все кандидаты имеют конфликты, Workflow не отправляет никаких задач

## Поток выполнения

```
1. Запрос URL загрузки
   └── POST в API MinerU для получения batch_id и upload_url

2. Загрузка файла
   └── Двоичная загрузка PDF-файла

3. Опрос результатов
   └── Повторные запросы до завершения обработки или сбоя
       └── Интервал: 2 секунды

4. Загрузка результатов
   └── Загрузка bundle (zip-формат)

5. Локальная материализация
   └── Извлечение bundle
       └── Извлечение содержимого Markdown
       └── Извлечение изображений
       └── Перепись путей к изображениям в Markdown на локальные относительные пути
       └── Запись в тот же каталог, что и PDF
```

## Выходы

### 1. Файл Markdown

- **Расположение**: Тот же каталог, что и PDF
- **Именование**: `<исходное имя файла>.md`
- **Содержимое**: Разобранный текст Markdown
- **Кодировка**: UTF-8

### 2. Каталог изображений

- **Расположение**: Тот же каталог, что и PDF: `Images_<ItemKey>/`
- **Содержимое**: Файлы изображений, извлечённые из PDF

### 3. Связанное вложение

- **Тип**: Ссылка на локальный файл
- **Расположение**: Под родительским элементом
- **Цель**: Файл `.md`

### Логика очистки

- Если `Images_<ItemKey>/` уже существует в целевом каталоге, старый каталог удаляется перед записью
- Избегает создания дублирующих связанных вложений `.md`, которые уже существуют

## Оценочная продолжительность

| Размер PDF | Оценочное время |
|---------|---------|
| Короткая статья (≤15 страниц) | 30 секунд - 1 минута |
| Стандартная (15-40 страниц) | 1-2 минуты |
| Длинная статья (40+ страниц) | 2-3 минуты |

Продолжительность в основном зависит от скорости обработки сервиса MinerU.

## Параметры

Workflow MinerU не имеет настраиваемых пользователем параметров.

## Рекомендации по модели

Модель LLM не требуется. Этот Workflow только вызывает сервис MinerU через HTTP API.

## Зависимости

- **Бэкенд**: Сервис MinerU (бэкенд Generic HTTP)
- **Конфигурация бэкенда**: Настройте бэкенд типа Generic HTTP в Менеджере бэкендов
- **Аутентификация**: Требуется действительный API-токен (токен Bearer)
- **URL сервиса MinerU**: `https://mineru.net` или другой развёрнутый экземпляр
