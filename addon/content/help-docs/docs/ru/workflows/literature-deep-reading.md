# Глубокое чтение

## Назначение

Выполните глубокое чтение статьи, генерируя структурированный аналитический обзор понимания чтения с множественных перспектив. Автоматически извлекает структуру глав, основные концепции и ссылки, поддерживает поабзацный перевод и выводит автономный HTML-документ для чтения.

## Сценарии использования

- Систематическое глубокое чтение важной статьи
- Получение комплексного анализа, включая аннотации глав, ключевые концепции и дальнейшее чтение
- Необходимость двуязычного параллельного чтения (оригинальный текст + перевод на целевой язык)

## Ограничения ввода

| Тип ограничения | Описание |
|---------|------|
| Единица ввода | Вложение |
| Принимаемые типы | `text/markdown`, `text/x-markdown`, `text/plain`, `application/pdf` |
| Лимит на родителя | Максимум 1 вложение |

### Способы запуска

- Прямой выбор вложения PDF или Markdown
- Выбор родительского элемента, и плагин автоматически раскроет его первое подходящее вложение

## Поток выполнения

Workflow Глубокого чтения — это **полностью автоматический** многоэтапный конвейер обработки, не требующий вмешательства от пользователя:

## Оценочная продолжительность

| Размер файла | Оценочное время |
|---------|---------|
| Короткая статья (≤10 страниц) | 8-12 минут |
| Стандартная (10-30 страниц) | 12-18 минут |
| Длинная статья (30+ страниц) | 18-25 минут |

Этот Workflow включает многоэтапную обработку (руководство → обогащение → перевод → организация → рендеринг), что делает его самым длительным Workflow анализа одной статьи.

## Рекомендации по модели

🟡 Рекомендуются модели с **сильным пониманием текста**. Этот Workflow требует многоуровневого глубокого анализа статьи (структура, концепции, логика аргументации), что предъявляет высокие требования к семантическому пониманию модели. Если доступна возможность делегирования подагентов, этапы могут выполняться параллельно, значительно сокращая общее время.

## Выходы

```
1. Фаза подготовки
   └── Загрузка исходного файла, генерация source_bundle.zip
       └── Содержит оригинальный текст, изображения и существующие ссылки

2. Руководство и сбор контекста
   └── Анализ структуры оригинального текста и метаданных
       └── Сбор связанного контекста через Host Bridge

3. Обогащение чтения
   └── Генерация аннотаций глав, ключевых концепций, анализа ссылок
       └── Обзор резюме и дальнейшего чтения

4. Поблочный перевод
   └── Нормализация перевода по стабильным блокам
       └── Генерация двуязычного параллельного перевода

5. Окончательный рендеринг
   └── Интеграция всех аналитических обзоров
       └── Рендеринг как автономный HTML-файл
```

## Выходные артефакты

После завершения выполнения связанное вложение, указывающее на сгенерированный HTML-файл, создаётся под родительским элементом:

- **Формат**: Автономный HTML-файл (может быть открыт в браузере)
- **Содержимое**: Полный обзор глубокого чтения, включающий структуру оригинального текста, аннотации глав, анализ концепций, ссылки, двуязычные переводы и т.д.
- **Жизненный цикл**: Каждый запуск перезаписывает и обновляет

## Параметры

| Параметр | Тип | Описание | По умолчанию |
|------|------|------|--------|
| `target_language` | string | Целевой язык | `zh-CN` |

Доступные значения: `zh-CN`, `en-US`, `ja-JP`, `ko-KR`, `de-DE`, `fr-FR`, `es-ES`, `ru-RU`. Также поддерживается пользовательский ввод.

## Зависимости

- **Бэкенд**: Бэкенд ACP (требуется поддержка протокола ACP)
- **Конфигурация бэкенда**: Настройте бэкенд типа ACP в Менеджере бэкендов
