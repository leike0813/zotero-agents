# Экспорт/Импорт заметок

## Назначение

Экспорт и импорт трёх типов структурированных заметок, сгенерированных `literature-analysis` (дайджест, ссылки, анализ цитирований), облегчая миграцию между экземплярами Zotero.

:::info Редактирование результатов анализа
Заметки, сгенерированные [Анализом литературы](#doc/workflows%2Fliterature-analysis), **рендерятся** из данных бэкенда; прямое изменение содержимого заметки не изменит данные бэкенда. Если вам нужно отредактировать результаты анализа, правильный подход: **Экспорт заметок** → изменить экспортированные файлы → использовать **Импорт заметок** для повторного импорта.
:::

## export-notes (Экспорт заметок)

### Сценарии использования

- Поделиться результатами анализа литературы с коллегами
- Импортировать результаты анализа в другой экземпляр Zotero
- Резервное копирование артефактов анализа литературы

### Ограничения ввода

| Тип ограничения | Описание |
|---------|------|
| Единица ввода | Родительский элемент |
| Метод выбора | Поддерживает смешанный выбор родительских элементов и трёх типов заметок |
| Поведение множественного выбора | Отображается только один диалог выбора каталога экспорта для множественного выбора |

### Экспортированные артефакты

| Файл | Описание |
|------|------|
| `digest.md` | Markdown дайджеста литературы |
| `references.json` | JSON списка ссылок |
| `citation_analysis.json` | JSON данных анализа цитирований |
| `citation_analysis.md` | Markdown отчёта анализа цитирований |
| `representative_image.jpg` | Репрезентативное изображение (когда заметка дайджеста содержит встроенное изображение) |

Репрезентативное изображение вставляется в `digest.md` как блок комментариев Markdown `zs:representative-image:v1`, ссылающийся с использованием относительного пути к тому же каталогу. Сбой экспорта изображения не блокирует экспорт текстовых и JSON-артефактов.

## Оценочная продолжительность

Завершается за секунды (чисто локальные файловые операции, бэкенд не требуется).

## import-notes (Импорт заметок)

### Сценарии использования

- Восстановление результатов анализа литературы в другом экземпляре Zotero
- Импорт артефактов анализа, которыми поделились коллеги

### Ограничения ввода

| Тип ограничения | Описание |
|---------|------|
| Единица ввода | Одиночный родительский элемент |
| Метод импорта | Выберите каталог, содержащий экспортированные артефакты |

### Поток импорта

```
1. Выбор каталога импорта
   └── Каталог должен содержать digest.md, references.json, citation_analysis.json

2. Проверка структуры
   └── references.json и citation_analysis.json проходят проверку структуры перед тем, как стать кандидатами
       └── Сбой проверки показывает предупреждение, но не блокирует импорт других артефактов

3. Разбор изображения
   └── Если digest.md содержит блок маркера zs:representative-image:v1
       └── Автоматический разбор репрезентативного изображения из того же каталога
       └── Пользователь также может вручную выбрать или очистить репрезентативное изображение

4. Запись
   └── Создание/обновление соответствующих заметок под родительским элементом
```

Сбой импорта изображения не блокирует импорт заметки дайджеста.

## Зависимости

- Подключение к бэкенду не требуется
- Зависит только от локального хранилища Zotero
