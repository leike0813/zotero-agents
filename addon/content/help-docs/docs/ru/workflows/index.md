# Обзор Workflow

## Что такое Workflow?

Workflow — это основная функция Zotero Agents, позволяющая комбинировать несколько шагов навыков в автоматизированные конвейеры обработки. Workflow определяет полную задачу: от получения ввода, обработки данных до создания вывода.

## Структура Workflow

```
workflow.json (файл манифеста)
├── manifest: объявляет метаданные, версию, название
├── parameters: определяет настраиваемые параметры
├── inputs: определяет типы ввода (вложения, элементы, заметки и т.д.)
├── hooks: скрипты хуков JavaScript (фильтрация вводов, создание запросов, применение результатов)
└── provider: указывает требуемый тип бэкенда
```

### Типы единиц ввода

| Тип | Описание |
|------|------|
| `attachment` | Файлы вложений элемента |
| `parent` | Родительский элемент выбранного элемента |
| `note` | Элемент заметки |
| `workflow` | Область пакетной обработки |

### Система хуков

Workflow могут запускать пользовательские скрипты JavaScript на различных этапах выполнения:

- **filterInputs**: Фильтрация и выбор вводов
- **buildRequest**: Создание содержимого запроса, отправляемого бэкенду
- **normalizeSettings**: Нормализация пользовательских настроек
- **applyResult**: Применение результатов, возвращённых бэкендом, к Zotero

## Три бэкенда выполнения

Workflow могут выполняться через три типа бэкендов:

| Бэкенд | Тип запроса | Сценарий использования |
|---------|-------------|---------|
| **Skill-Runner** | `skill.run.v1` | Общее выполнение навыков, поддерживает интерактивный режим |
| **ACP** | `acp.skill.run.v1` | Выполнение навыков через бэкенд ACP |
| **Generic HTTP** | `generic-http.request.v1` | Вызовы HTTP API |

## Официальный пакет Workflow

Официальные Workflow публикуются и устанавливаются как **автономные пакеты**, отделённые от самого плагина. Способы установки:

- Контекстное меню → **Zotero Agents** → **📦 Установить официальный пакет Workflow**
- Нажмите **Установить официальный пакет Workflow** в Настройках

Официальные пакеты поддерживают три канала обновлений: stable / beta / dev. Плагин автоматически проверяет обновления при запуске.
