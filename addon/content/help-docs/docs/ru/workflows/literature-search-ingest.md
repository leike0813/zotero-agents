# Поиск и приём литературы

## Назначение

Поиск академической литературы через ИИ и приём результатов напрямую в Zotero. Поддерживает несколько режимов поиска с интерактивным подтверждением перед выполнением операции приёма.

## Сценарии использования

- Поиск и пакетный приём соответствующей литературы при исследовании новой темы
- Ввод названия, DOI, arXiv ID или PMID известной статьи для быстрого импорта
- Расширение поиска связанной литературы на основе базовой статьи

## Ограничения ввода

| Тип ограничения | Описание |
|---------|------|
| Единица ввода | workflow (не нужно выбирать элементы) |
| Способ запуска | Запуск из контекстного меню или Панели мониторинга, не нужно предварительно выбирать элементы |

## Режимы поиска

| Режим | Описание |
|------|------|
| `auto` | Автоматическое определение наиболее подходящего режима поиска (по умолчанию) |
| `topic_expansion` | Поиск по исследовательскому направлению или теме для поиска связанной литературы |
| `paper_seed_expansion` | Расширение поиска на основе базовой статьи |
| `targeted_ingest` | Точный поиск и приём одной статьи |

## Поток выполнения

```
1. Фаза подтверждения плана
   └── Чтение библиотеки Zotero и контекста Synthesis
       └── Автоматическое определение режима поиска (автоматический режим)
       └── Представление плана поиска пользователю
       └── Ожидание подтверждения от пользователя

2. Фаза поиска (без приёма)
   └── Поиск литературы-кандидатов согласно подтверждённому плану
       └── Отображение списка результатов поиска
       └── Пользователь выбирает литературу для приёма

3. Фаза приёма
   └── Приём статей по одной через zotero-bridge
       └── Включает импорт метаданных и импорт вложений PDF
       └── Отображение прогресса приёма

4. Завершение
   └── Вывод сводки результатов приёма
       └── Включает информацию об успешных/неудачных элементах
```

### Детали взаимодействия

- Этот Workflow выполняется в **интерактивном** режиме, требуя подтверждения от пользователя в ключевых точках
- Подтверждение плана: После того как ИИ представляет план поиска, пользователь подтверждает или корректирует его
- Подтверждение списка: После отображения результатов поиска пользователь отмечает элементы для приёма
- Прогресс выполнения можно отслеживать в Панели мониторинга

## Рекомендации по модели

🔴 **Должна** иметь возможность веб-поиска. Суть этого Workflow — поиск академической литературы онлайн — модели без возможности веб-поиска не могут выполнить эту задачу.
🟢 Способность модели к рассуждению не должна быть сильной — поиск и приём по сути являются задачами поиска и вызова инструментов, с которыми могут справиться лёгкие модели.

## Выходы

- Результаты поиска принимаются напрямую как элементы Zotero
- Автоматически пытается загрузить вложения PDF (лучшие усилия)
- Можно указать целевую Коллекцию для категоризации

## Параметры

| Параметр | Тип | Описание | По умолчанию |
|------|------|------|--------|
| `query` | string | Тема поиска, исследовательское направление, название статьи, DOI, arXiv ID, PMID и т.д. | — |
| `searchMode` | string | Режим поиска | `auto` |
| `targetCollection` | string | Целевая Коллекция (необязательно) | Пусто |

### Доступные значения searchMode

- `auto`: Автоматическое определение
- `topic_expansion`: Расширение темы
- `paper_seed_expansion`: Расширение базовой статьи
- `targeted_ingest`: Целенаправленный приём

## Зависимости

- **Бэкенд**: Бэкенд ACP (требуется поддержка протокола ACP)
- **Навык**: Навык `literature-search-ingest` должен быть развёрнут на бэкенде
