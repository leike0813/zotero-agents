# レビューハブ

Review サーフェスは、Synthesis システムの全保留中レビューアイテムを一元管理する場所である。**引用マッチ**、**概念**、**トピックグラフ**の 3 つのサブタブで構成される。

## 引用マッチレビュー

システムが参考文献と Zotero アイテムを自動的にマッチングする際、確実性を判断できないマッチは提案としてレビューキューに提出される。

### マッチ提案のステータス

| ステータス | 説明 |
|-----------|------|
| **保留中** | システムが生成したマッチ候補。ユーザーの確認または却下を待っている |
| **承認済み** | ユーザーがバインディングを確認。参考文献が Zotero アイテムにリンクされた |
| **却下済み** | ユーザーがバインディングを拒否した |
| **再オープン** | 以前に処理された提案がレビューのために再オープンされた |

### 利用可能なアクション

- **承認**: 引用-アイテムのバインディング関係を確認
- **却下**: マッチ提案を拒否
- **一括操作**: 複数の提案を選択して一括承認または却下

### マッチ信頼度

信頼度レベルの説明については [インデックスと引用グラフ](#doc/synthesis%2Findex-and-citation) を参照。決定論的および高信頼度のマッチは通常自動的に処理され、中信頼度以下のマッチがレビューキューに入る。

### フィルタリングと並び替え

提案リストは以下の基準でフィルタリングできる。

- マッチステータス（保留中/承認済み/却下済み）
- マッチ戦略（DOI/タイトル/著者など）
- 信頼度レベル
- 時刻または関連性による並び替え

## 概念レビュー

概念知識ベースの自動拡張により、低信頼度の概念マッチ提案が生成される場合があり、ユーザーによるレビューと確認が必要となる。

### レビュー対象

- **新概念提案**: 文献から自動的に抽出された新概念候補
- **意味の確認**: 既存の概念に新しい意味（sense）が追加される場合の確認
- **別名提案**: 同じ概念の代替名が検出された場合の確認

### 操作方法

各提案には、概念名、抽出元、信頼度レベル、根拠証拠が表示される。以下の操作が可能。

- **承認**: 提案を確認し、概念 KB に書き込む
- **却下**: 提案を却下する
- **コンテキスト表示**: 概念が文献内でどこに出現しているかを確認

## トピックグラフレビュー

システムがトピック間の潜在的な関係を検出すると、レビュー用の関係提案が生成される。

### 関係タイプ

| 関係 | 説明 |
|------|------|
| `broader_than` | A は B よりも広範なトピックである |
| `related_to` | 2 つのトピックが関連している |
| `overlaps_with` | 2 つのトピックに内容の重複がある |
| `contrasts_with` | 2 つのトピックが互いに対照的である |

### 提案内容

各提案には以下が表示される。

- **ソースとターゲットのトピック**名と説明
- **提案される関係タイプ**
- **信頼度**（トピック内容のセマンティック分析に基づく）
- **根拠証拠**（共同カバー論文など）

### 操作方法

- **承認**: 関係を確認し、トピックグラフに書き込む
- **却下**: 関係提案を却下する
- **再オープン**: 以前に処理された提案をレビューのために再オープンする
