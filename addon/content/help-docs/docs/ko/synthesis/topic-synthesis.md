# 토픽 통합 생성

## 토픽 통합이란?

토픽 통합은 관련된 문헌 그룹을 체계적으로 분석하고 통합하는 프로세스입니다. AI Workflow를 통해 자동으로 핵심 정보를 추출하고, 토픽 구조를 식별하며, 포괄적인 분석 보고서를 생성합니다.

## Topics 서피스

Synthesis Workbench → Topics 페이지에서 생성된 모든 토픽을 탐색하고 관리할 수 있습니다. Topics 서피스는 **세 가지 뷰 모드**를 지원합니다:

| 뷰 | 설명 | 사용 사례 |
|------|-------------|----------|
| **그래프 뷰** | 토픽을 노드로, 관계를 엣지로 하는 힘 방향 그래프 | 토픽 간 연관을 직관적으로 이해 |
| **그리드 뷰** | 제목, 논문 수, 요약 및 작업 버튼이 있는 카드 | 토픽 탐색 및 찾기 |
| **리스트 뷰** | 다음 열이 있는 테이블 뷰: 이름, 논문 수, 생성 시간, 업데이트 날짜, 상태 | 정렬 및 일괄 작업 |

### 토픽 관리 작업

- **검색**: 토픽 이름 및 설명으로 검색
- **정렬**: 제목, 논문 수 또는 업데이트 날짜 기준으로 정렬
- **새 토픽 생성**: 생성 버튼을 클릭하여 Workflow 파이프라인 시작
- **토픽 업데이트**: 파이프라인을 다시 실행하여 토픽 분석 업데이트
- **토픽 삭제**: 더 이상 필요하지 않은 토픽 제거

## 생성 프로세스

토픽 생성은 Workflow에 의해 구동되며 여러 단계의 자동화된 파이프라인입니다:

```
1. create-topic-prepare
   → 문헌 데이터 수집, 논문 세트 구축
   
2. topic-synthesis-core-enrichment
   → 핵심 풍부화: 정보 추출, 지식 연관
   
3. topic-synthesis-finalize
   → 최종 분석 아티팩트 및 보고서 생성

(update-topic-synthesis-prepare는 기존 토픽 업데이트에 사용됨)
```

### 사전 요구 사항

- [Skill-Runner 백엔드](#doc/backends%2Fskill-runner) 구성됨
- 라이브러리의 관련 논문
- 논문에서 다이제스트 및 인용 분석이 생성됨 (선택 사항, 권장)

이 파이프라인은 [토픽 통합 생성](#doc/workflows%2Ftopic-synthesis) Workflow에 의해 조율됩니다.

## 토픽 인스펙터

토픽을 생성한 후 클릭하면 토픽 인스펙터로 진입합니다. 이것은 8개의 서브 페이지를 포함하는 멀티 페이지 리더로, 각 페이지는 토픽의 서로 다른 차원을 제시합니다.

### Overview

- 토픽 이름, 설명, 중요도 점수
- 핵심 주장 요약
- 통계 (논문 수, 카테고리 수, 주장 수 등)
- 연관된 토픽 그래프 위치 정보

### Taxonomy

토픽의 계층적 분류 구조를 표시합니다:

- 더 넓은 토픽: 광범위한 토픽 영역
- 더 좁은 토픽: 더 구체적인 서브 토픽
- 관련 토픽: 다른 연관 토픽
- 토픽 그래프에서의 위치 및 계층

### Claims

문헌에서 추출된 핵심 주장 및 단언:

- 각 주장에는 원본 증거 인용이 포함됨
- 주장이 유래된 논문을 표시
- 주장 유형 (발견 / 가설 / 결론 등)
- 주장을 지지하는 논문 수

### Compare

동일한 토픽에 대한 서로 다른 논문 간의 관점 비교:

- 비교 차원 (방법, 결론, 데이터셋 등)
- 각 논문의 입장 및 논거
- 합의와 분기의 시각화

### Future Directions

문헌 분석을 통해 식별된 연구 격차 및 미래 방향:

- 개방된 질문
- 잠재적 연구 방향
- 관련 과제 및 권장 사항

### Coverage

토픽이 관련 문헌을 다루는 정도를 분석합니다:

- 토픽이 다루는 논문 리스트
- 논문 완전성 (다이제스트/인용 분석 아티팩트 존재 여부)
- 다루어진 측면과 다루어지지 않은 측면

### References

토픽과 연관된 모든 참고문헌, 바인딩 상세 정보 포함:

- 각 인용에 대한 Zotero 항목 링크
- 토픽에서의 인용 역할 (지지 / 대조 / 배경)
- 인용 출처 및 컨텍스트

### Report (전체 보고서)

생성된 구조화된 통합 분석 보고서 (Markdown 형식):

- 완전한 토픽 분석 텍스트
- Markdown 또는 자체 포함 HTML로 내보내기 가능
- 학술 작성에서 참고 자료로 사용하기 적합

## 토픽 그래프

토픽 그래프는 토픽 간의 관계를 보여주는 계층적 토픽 네트워크입니다:

### 노드 유형

| 유형 | 설명 |
|------|-------------|
| **materialized** | 실제로 생성된 구조화된 토픽 |
| **placeholder** | 존재한다고 추론되지만 아직 생성되지 않은 토픽 플레이스홀더 |

### 엣지 상태

| 상태 | 설명 |
|--------|-------------|
| `suggested` | 시스템이 제안한 관계 (리뷰 대기 중) |
| `confirmed` | 사용자가 확인한 관계 |
| `rejected` | 사용자가 거절한 관계 |
| `stale` | 오래된 데이터, 재평가 대기 중 |
| `deleted` | 삭제된 관계 |

### 관계 유형

| 관계 | 설명 |
|--------------|-------------|
| `broader_than` | A가 B보다 더 넓은 토픽 |
| `related_to` | 두 토픽이 관련됨 |
| `overlaps_with` | 두 토픽이 중복됨 |
| `contrasts_with` | 두 토픽이 상호 대비됨 |

### 토픽 관리

- **새 토픽 생성**: Topics 페이지에서 "생성" 클릭
- **토픽 편집**: 이름, 설명, 중요도 등 수정
- **논문 연관**: 토픽에서 논문을 추가 또는 제거
- **토픽 그래프 탐색**: 토픽 간의 관계 네트워크 보기
