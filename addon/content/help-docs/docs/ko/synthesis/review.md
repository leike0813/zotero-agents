# 리뷰 허브

리뷰 서피스는 Synthesis 시스템의 모든 대기 중인 리뷰 항목을 처리하는 중앙 장소입니다. **인용 매치**, **개념**, **토픽 그래프**의 세 가지 서브 탭으로 구성됩니다.

## 인용 매치 리뷰

시스템이 참고문헌을 Zotero 항목과 자동으로 매칭할 때, 확실하지 않은 매치는 제안 형태로 리뷰 큐에 제출됩니다.

### 매치 제안 상태

| 상태 | 설명 |
|--------|-------------|
| **대기 중** | 사용자의 확인 또는 거절을 기다리는 시스템 생성 매치 후보 |
| **수락됨** | 사용자가 바인딩을 확인; 참고문헌이 이제 Zotero 항목과 연결됨 |
| **거절됨** | 사용자가 바인딩을 거절함 |
| **재개됨** | 이전에 처리된 제안이 리뷰를 위해 다시 열림 |

### 사용 가능한 작업

- **수락**: 인용-항목 바인딩 관계를 확인
- **거절**: 매치 제안을 거부
- **일괄 작업**: 여러 제안을 선택하여 일괄 수락 또는 거절

### 매치 신뢰도

신뢰도 수준에 대한 설명은 [인덱스 & 인용 그래프](#doc/synthesis%2Findex-and-citation)를 참조하세요. 결정적(deterministic) 및 고신뢰도 매치는 일반적으로 자동으로 처리되며, 중간 이하 신뢰도 매치는 리뷰 큐로 진입합니다.

### 필터링 및 정렬

다음 기준으로 제안 리스트를 필터링할 수 있습니다:

- 매치 상태 (대기 중 / 수락됨 / 거절됨)
- 매치 전략 (DOI / 제목 / 저자 등)
- 신뢰도 수준
- 시간 또는 관련성 기준 정렬

## 개념 리뷰

개념 지식 베이스의 자동 확장은 신뢰도가 낮은 개념 매치 제안을 생성할 수 있으며, 사용자의 리뷰와 확인이 필요합니다.

### 리뷰 대상

- **새 개념 제안**: 문헌에서 자동 추출된 새 개념 후보
- **의미 확인**: 기존 개념에 새 의미(sense)가 추가될 때 확인
- **별칭 제안**: 동일한 개념에 대한 대체명이 감지될 때 확인

### 작업 방법

각 제안에는 개념 이름, 추출 출처, 신뢰도 수준 및 뒷받침하는 근거가 표시됩니다. 다음 작업을 수행할 수 있습니다:

- **수락**: 제안을 확인하고 개념 KB에 기록
- **거절**: 제안을 무시
- **컨텍스트 보기**: 개념이 문헌에서 나타나는 위치 확인

## 토픽 그래프 리뷰

시스템이 토픽 간의 잠재적 관계를 감지하면, 리뷰를 위한 관계 제안이 생성됩니다.

### 관계 유형

| 관계 | 설명 |
|--------------|-------------|
| `broader_than` | A가 B보다 더 넓은 토픽 |
| `related_to` | 두 토픽이 관련됨 |
| `overlaps_with` | 두 토픽이 내용상 중복됨 |
| `contrasts_with` | 두 토픽이 상호 대비됨 |

### 제안 내용

각 제안에는 다음이 표시됩니다:

- **출발 및 대상 토픽** 이름 및 설명
- **제안된 관계 유형**
- **신뢰도** (토픽 내용의 의미 분석 기반)
- **근거 증거** (공동 참조 논문 등)

### 작업 방법

- **수락**: 관계를 확인하고 토픽 그래프에 기록
- **거절**: 관계 제안을 무시
- **재개**: 이전에 처리된 제안을 리뷰를 위해 다시 열기
