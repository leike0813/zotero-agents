# Literature Search & Ingest

## 목적

AI를 통해 학술 문헌을 검색하고 결과를 Zotero로 직접 수집합니다. 여러 검색 모드를 지원하며, 수집 작업을 실행하기 전에 대화형 확인을 거칩니다.

## 사용 사례

- 새로운 주제를 연구할 때 관련 문헌을 검색하고 일괄 수집할 때
- 알려진 논문의 제목, DOI, arXiv ID 또는 PMID를 입력하여 빠르게 가져올 때
- 시드 논문을 기반으로 관련 문헌 검색을 확장할 때

## 입력 제약

| 제약 유형 | 설명 |
|---------|------|
| 입력 단위 | workflow (항목을 선택할 필요 없음) |
| 실행 방법 | 컨텍스트 메뉴 또는 대시보드에서 실행, 항목을 미리 선택할 필요 없음 |

## 검색 모드

| 모드 | 설명 |
|------|------|
| `auto` | 가장 적합한 검색 모드를 자동 결정합니다 (기본값) |
| `topic_expansion` | 연구 방향 또는 주제별로 검색하여 관련 문헌을 찾습니다 |
| `paper_seed_expansion` | 시드 논문을 기반으로 검색을 확장합니다 |
| `targeted_ingest` | 단일 논문을 정밀하게 위치하고 수집합니다 |

## 실행 흐름

```
1. 계획 확인 단계
   └── Zotero 라이브러리 및 통합 컨텍스트 읽기
       └── 검색 모드 자동 결정 (auto 모드)
       └── 검색 계획을 사용자에게 제시
       └── 사용자 확인 대기

2. 검색 단계 (수집 없음)
   └── 확인된 계획에 따라 후보 문헌 검색
       └── 검색 결과 목록 표시
       └── 사용자가 수집할 문헌 선택

3. 수집 단계
   └── zotero-bridge를 통해 논문을 하나씩 수집
       └── 메타데이터 가져오기 및 PDF 첨부파일 가져오기 포함
       └── 수집 진행 상황 표시

4. 완료
   └── 수집 결과 요약 출력
       └── 성공/실패 항목 정보 포함
```

### 상호작용 세부 사항

- 이 Workflow는 **인터랙티브** 모드로 실행되며, 주요 지점에서 사용자 확인이 필요합니다
- 계획 확인: AI가 검색 계획을 제시한 후 사용자가 확인하거나 조정합니다
- 목록 확인: 검색 결과가 표시된 후 사용자가 수집할 항목을 확인합니다
- 실행 진행 상황은 대시보드에서 모니터링할 수 있습니다

## 모델 권장 사항

🔴 웹 검색 기능이 **반드시** 필요합니다. 이 Workflow의 핵심은 온라인에서 학술 문헌을 검색하는 것입니다 — 웹 검색 기능이 없는 모델은 이 작업을 수행할 수 없습니다.
🟢 모델의 추론 능력은 강력할 필요가 없습니다 — 검색 및 수집은 본질적으로 검색 및 도구 호출 작업이며, 경량 모델로도 처리할 수 있습니다.

## 출력

- 검색 결과가 Zotero 항목으로 직접 수집됩니다
- PDF 첨부파일 다운로드를 자동으로 시도합니다 (최선 노력)
- 분류를 위해 대상 컬렉션을 지정할 수 있습니다

## 파라미터

| 파라미터 | 유형 | 설명 | 기본값 |
|------|------|------|--------|
| `query` | string | 검색 주제, 연구 방향, 논문 제목, DOI, arXiv ID, PMID 등 | — |
| `searchMode` | string | 검색 모드 | `auto` |
| `targetCollection` | string | 대상 컬렉션 (선택 사항) | 비어 있음 |

### searchMode 사용 가능한 값

- `auto`: 자동 결정
- `topic_expansion`: 주제 확장
- `paper_seed_expansion`: 시드 논문 확장
- `targeted_ingest`: 대상 수집

## 의존성

- **스킬**: `literature-search-ingest` 스킬이 백엔드에 배포되어야 합니다
