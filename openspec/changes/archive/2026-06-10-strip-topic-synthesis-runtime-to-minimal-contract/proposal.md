# Change: Strip Topic Synthesis Runtime to Minimal Contract

## Why

The split topic synthesis runtime currently carries an audit-heavy contract:
artifact registries, action receipts, gate/action transcript files, payload/file
hashes, digest freshness checks, and large diagnostics. These fields increase
token usage, enlarge the failure surface, and have already caused valid topic
synthesis runs to fail apply because `content_hash` and Host `payload_hash`
were confused.

The topic synthesis workflow is not a security or banking system. Its practical
requirement is simpler: the business flow must execute, agent-authored payloads
must be schema-checked, and runtime-owned outputs must be generated by runtime
commands rather than accepted directly from the agent.

## What Changes

- Reduce the split runtime state to stage progression and the minimum context
  needed by later stages.
- Stop generating long-lived audit artifacts such as `artifact-registry.json`,
  `runtime/gate-transcript/`, `runtime/action-transcript/`, and
  `runtime/stage-receipts/`.
- Remove `gate.py --action audit`.
- Remove hash fields from split runtime output contracts and generated
  `SKILL.md` guidance.
- Stop blocking Host apply on stale or mismatched digest hashes.
- Keep only core checks: allowed stage, payload schema, required files, final
  result kind/operation, complete section files, and non-embedded digest refs.

## Impact

This is intentionally a breaking simplification for new split topic synthesis
runtime artifacts. Legacy monolithic skill packages and already persisted topic
artifacts remain readable, but the split suite no longer treats hash/audit data
as a public contract.
