# Вкладка SkillRunner

Вкладка SkillRunner используется для просмотра и взаимодействия с запусками, выполняемыми через бэкенд Skill-Runner. В отличие от Навыков ACP, которые фокусируются на одноразовом выполнении навыков, вкладка SkillRunner подчёркивает управление интерактивными сессиями.

## Обзор интерфейса

![Панель SkillRunner](/img/docs/sidebar/skillrunner-tab.png)

```
┌─────────────────────────────────────┐
│  Баннер: Заголовок / requestId / Статус    │
├─────────────────────────────────────┤
│  ← Выдвижная панель задач  │  Основная область контента   │  Детали → │
│               │  Просмотр транскрипта            │
│  Выполняется  │  Компонент плана             │
│  └─ бэкенд1  │  Компонент промпта           │
│     └─ задача A │  Область ответа                 │
│  Завершено    │                             │
│  └─ бэкенд1  │                             │
│     └─ задача B │                             │
└─────────────────────────────────────┘
```

## Баннер

Баннер отображает информацию о текущей выбранной задаче:

- **Заголовок**: Название задачи или идентификатор навыка
- **Request ID**: Уникальный идентификатор запроса для задачи
- **Статус**: Статус выполнения (выполняется / waiting_user / waiting_auth / завершено / ошибка)
- **Бэкенд**: Информация о бэкенде
- **Движок**: Используемый движок (например, gemini, claude и т.д.)
- **Модель**: Используемая модель
- **Обновлено**: Время последнего обновления
- **Кнопка отмены задачи**

## Выдвижная панель задач (слева)

Левая выдвижная панель отображает все задачи SkillRunner, разделённые на группы Выполняется и Завершено. Каждая запись задачи показывает сводную информацию, индикатор статуса и действие архивирования. Нажмите на запись для переключения на детальный вид этой задачи.

## Основная область контента

### Просмотр транскрипта

Просмотр транскрипта SkillRunner использует **модель чата мышления**, которая интеллектуально обрабатывает непрерывное рассуждение:

- **Блоки мышления**: Процесс рассуждения ИИ отображается как отдельные блоки мышления
- **Вызовы инструментов**: Показывает название инструмента, сводку ввода и статус выполнения
- **Сообщения**: Сообщения диалога ассистента и пользователя
- **Ревизия**: Записи изменений версий вывода

Также поддерживает режимы отображения **Plain / Bubble**.

### Поток аутентификации

Вкладка SkillRunner поддерживает потоки аутентификации, позволяя завершить аутентификацию бэкенда, не выходя из панели:

**Триггеры аутентификации:**

- Автоматически запускается при запуске навыка, требующего аутентификации
- Компонент промпта отображает запрос аутентификации

**Поддерживаемые методы аутентификации:**

| Метод | Описание | Сценарии использования |
|------|------|---------|
| **OAuth прокси** | Завершите поток OAuth через браузер | Рекомендуемый метод, для движков, поддерживающих OAuth |
| **Ввод кода аутентификации** | Вручную введите код аутентификации или URL | Когда движок сгенерировал ссылку аутентификации |
| **Импорт файла** | Импортируйте файл учётных данных | Когда файл учётных данных уже доступен |
| **Встроенный TUI** | Запустите терминал напрямую в панели | Когда требуется интерактивный вход |

**Пример потока аутентификации (OAuth):**

1. Запуск обнаруживает, что требуется аутентификация
2. Компонент промпта показывает "Требуется аутентификация" и доступные методы аутентификации
3. Пользователь выбирает прокси OAuth
4. Браузер открывает страницу OAuth
5. Пользователь завершает аутентификацию
6. Запуск автоматически возобновляет выполнение

### Компонент промпта

| Статус | Отображаемое содержимое |
|------|---------|
| `waiting_user` | Ожидание ввода от пользователя; показывает описание контекста и быстрые опции (если доступны) |
| `waiting_auth` | Ожидание аутентификации; показывает выбор метода аутентификации и ввод |
| `running` | Индикатор выполнения |
| `completed` | Подтверждение статуса завершения |
| `error` | Информация об ошибке и предложения по устранению неполадок |

### Область ответа

- **Поле ввода текста**: Введите содержимое ответа
- **Кнопка Отправить/Отменить**

В отличие от Навыков ACP, область ответа вкладки SkillRunner не имеет селекторов режима/модели/рассуждения (они настраиваются в настройках бэкенда).

## Выдвижная панель деталей (справа)

| Область | Содержимое |
|------|------|
| **Метаданные запуска** | Заголовок, requestId, taskKey, статус, флаги терминала/ожидания |
| **Информация о бэкенде** | бэкенд, движок, модель |
| **Время обновления** | Время последней активности |
| **Информация о взаимодействии** | Текущая ожидающая информация о взаимодействии (если есть) |
| **Сводка сессии** | Сводка исторической сессии |
| **Сводка ревизии** | Записи изменений версий вывода |

## Связанная конфигурация

Перед использованием вкладки SkillRunner должен быть настроен [бэкенд Skill-Runner](../backends/skill-runner).
