# Навыки ACP

Вкладка Навыки ACP используется для мониторинга и управления запусками навыков, выполняемых через бэкенд ACP. В отличие от непрерывного диалога ACP Чата, Навыки ACP предназначены для одноразовых или периодически выполняемых задач навыков.

## Обзор интерфейса

Панель Навыки ACP разделена на следующие основные области:

![Панель Навыки ACP](/img/docs/sidebar/acp-skills.png)

```
┌─────────────────────────────────────┐
│  Баннер: Название задачи / Статус / Бэкенд   │
├─────────────────────────────────────┤
│  ← Выдвижная панель запусков  │  Основная область контента  │  Детали → │
│               │  Просмотр транскрипта            │
│  Выполняется  │  Компонент плана             │
│  └─ бэкенд1  │  Компонент промпта           │
│     ├─ запуск A  │  Область ответа                 │
│     └─ запуск B  │                             │
│  Завершено    │                             │
│  └─ бэкенд1  │                             │
│     └─ запуск C  │                             │
└─────────────────────────────────────┘
```

## Баннер

Область Баннера отображает мета-информацию и кнопки действий для текущего выбранного запуска:

- **Название задачи**: Название навыка запуска
- **Статус**: Индикатор статуса запуска (выполняется / завершено / ошибка / отменено и т.д.)
- **Бэкенд**: Бэкенд ACP, выполняющий запуск
- **Кнопки действий**: Подключить/Отключить, Отменить задачу

## Выдвижная панель запусков (слева)

Левая выдвижная панель организует все запуски Навыков ACP в древовидной структуре:

### Группировка

| Группа | Описание |
|------|------|
| **Выполняется** | Текущие выполняющиеся задачи, сгруппированные по бэкендам |
| **Завершено** | Завершённые задачи, сгруппированные по бэкендам |

Каждая запись задачи отображает сводную информацию (ID навыка, статус, время) и имеет индикатор внимания (светодиод) для отметки изменений статуса. Нажмите на любую запись задачи для переключения на детальный вид этого запуска.

### Архивирование

Завершённые задачи могут быть удалены из списка с помощью кнопки архивирования (архивирование только скрывает их в текущей сессии и не влияет на записи запусков).

## Основная область контента

### Просмотр транскрипта

После выбора запуска основная область контента отображает полный транскрипт этого запуска, включая:

- **Сообщения**: Содержимое диалога ассистента и пользователя
- **Вызовы инструментов**: Инструменты, вызванные ИИ, и их результаты, показывающие название инструмента, сводку ввода и светодиод статуса
- **Процесс мышления**: Процесс рассуждения ИИ (если доступен)
- **События статуса**: Изменения состояния во время выполнения

Транскрипт поддерживает **Режим Plain** (сообщения раскрашены по ролям на левой границе) и **Режим Bubble** (сообщения в стиле пузырей, последовательные вызовы инструментов автоматически сворачиваются в группы), переключается с помощью кнопки в правом верхнем углу.

### Компонент плана

Когда запуск включает многошаговый план, компонент плана отображает текущий прогресс, завершённые шаги и ожидающие шаги, при этом каждый шаг имеет значок статуса (в процессе/завершено/ошибка).

### Компонент промпта

Компонент промпта отображает различные интерактивные подсказки в зависимости от статуса запуска:

| Статус | Отображаемое содержимое |
|------|---------|
| `waiting_user` | Подсказка, ожидающая ответа пользователя, с описанием контекста и опциями быстрого ответа |
| `permission` | Подсказка запроса разрешения, с предпросмотром команды и кнопками одобрить/отклонить |
| `disconnected` | Подсказка переподключения; нажмите для подключения |
| `running` | Индикатор выполнения |
| `completed` | Подтверждение статуса завершения |
| `error` | Информация об ошибке и предложения по устранению неполадок |

### Область ответа

Область ответа внизу содержит:

- **Поле ввода текста**: Введите содержимое ответа
- **Выбор режима** (необязательно): Переключатель режима выполнения
- **Выбор модели** (необязательно): Переключатель модели ИИ
- **Усилие рассуждения** (необязательно): Уровень усилия рассуждения
- **Кнопка Отправить/Отменить**
- **Счётчик использования**: Круговая диаграмма, показывающая использование токенов (использовано/лимит)
- **Подсказка горячих клавиш**: Горячая клавиша для отправки ответов

Черновики ответов сохраняются для каждого запроса — переключение запусков и переключение обратно сохраняет неотправленное содержимое.

## Выдвижная панель деталей (справа)

Правая выдвижная панель отображает подробную информацию о выбранном запуске со следующими сворачиваемыми областями:

| Область | Содержимое |
|------|------|
| **Путь запуска** | Рабочий каталог, пути к файлам результатов |
| **Информация о Runner** | бэкенды, агент, режим, модель, рассуждение, навык, сессия |
| **Информация о валидации** | Статус валидации, количество исправлений, детали ошибок |
| **Зависимости выполнения** | Список зависимостей среды выполнения |
| **Ревизия вывода** | История ревизий вывода |
| **Лог выполнения** | Записи лога во время выполнения |
| **Result JSON** | Окончательный структурированный вывод (расширяемый) |

## Обработка разрешений

Когда запуск требует разрешений записи Zotero или разрешений вызова инструмента ACP, компонент промпта отображает запрос разрешения:

- **Предпросмотр команды**: Показывает запрашиваемую операцию
- **Информация об источнике**: Кто инициировал запрос
- **Кнопки действий**: Одобрить / Отклонить
- Разверните для просмотра полных деталей запроса

## Связанная конфигурация

Панель Навыки ACP требует настройки [бэкенда ACP](../backends/acp) перед использованием.
