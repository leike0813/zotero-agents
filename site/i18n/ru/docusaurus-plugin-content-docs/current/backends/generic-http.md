# Конфигурация бэкенда Generic HTTP

## Назначение

Бэкенд Generic HTTP используется для отправки сырых HTTP-запросов на любой URL. Он не выполняет навыки агентов, а служит как универсальный HTTP-клиент.

## Основной сценарий использования: Разбор документов MinerU

Основное использование бэкенда Generic HTTP — поддержка **Workflow MinerU** — Workflow разбора PDF-документов.

MinerU — это сервис разбора документов, который конвертирует PDF-файлы в формат Markdown. Workflow MinerU отправляет запросы в сервис MinerU через бэкенд Generic HTTP для получения результатов разбора.

### Настройка MinerU

1. Посетите [mineru.net](https://mineru.net) для регистрации учётной записи и получите API-токен на странице **API → Управление API**
2. Откройте **Инструменты → [Менеджер бэкендов](backend-manager)**
3. Переключитесь на вкладку **Generic HTTP**
4. Нажмите **Добавить Generic HTTP**
5. Заполните:

| Поле | Значение |
|-------|-------|
| Отображаемое имя | `MinerU Official` |
| Base URL | `https://mineru.net` |
| Аутентификация | `bearer` |
| Токен авторизации | Вставьте ваш API-токен |
| Тайм-аут | `60000` (60 секунд) |

6. Нажмите **Сохранить** в правом нижнем углу

## Поля конфигурации

| Поле | Обязательно | Описание |
|-------|----------|-------------|
| Отображаемое имя | Да | Отображаемое имя для бэкенда |
| Base URL | Да | Базовый адрес HTTP-сервиса |
| Bearer Token | Нет | Токен аутентификации |
| Тайм-аут | Нет | Тайм-аут запроса (миллисекунды) |

## Технические детали

Бэкенд Generic HTTP поддерживает:
- **Одношаговые запросы**: `generic-http.request.v1` — Отправка одного HTTP-запроса
- **Многоступенчатые конвейеры**: `generic-http.steps.v1` — Цепочка запросов с извлечением JSON-пути (`$.*` выражения), извлечение значений из предыдущих ответов в качестве параметров для последующих запросов
- **Multipart загрузки**: Поддержка загрузки файлов
- Механизмы опроса и повторных попыток
