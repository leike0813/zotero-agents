# Обзор конфигурации бэкендов

Zotero Agents поддерживает три типа бэкендов, каждый из которых подходит для различных сценариев использования.

## Как выбрать

### 🥇 Первый выбор: Бэкенд ACP

Если у вас уже установлен любой совместимый с ACP инструмент агента на вашем компьютере (Codex, Claude Code, OpenCode, Hermes Agent, OpenClaw, Qwen Code и т.д.), вы можете использовать бэкенд ACP напрямую. **Никакой дополнительной нагрузки по настройке** — просто выберите соответствующего агента из списка пресетов в Менеджере бэкендов, и плагин автоматически управляет жизненным циклом процесса.

Некоторые агенты (такие как OpenCode и Codex) также поддерживают изоляцию каталогов конфигурации и каталогов постоянных сессий через переменные окружения, что упрощает управление несколькими рабочими контекстами.

→ [Конфигурация бэкенда ACP](./acp)

### 🥈 Второй выбор: Skill-Runner, развёрнутый в Docker

Если вам нужно **постоянное фоновое выполнение** (задачи продолжают выполняться после закрытия Zotero, и вы можете возобновить или получить результаты при следующем запуске), или у вас есть возможность настроить сервер в вашей локальной сети, рекомендуется развернуть Skill-Runner с помощью Docker в качестве постоянного сервиса.

Skill-Runner, развёрнутый в Docker, работает независимо от Zotero и поддерживает многопользовательский доступ, веб-интерфейс управления, управление движками и многое другое.

→ [Развёртывание и настройка Skill-Runner](./skill-runner)

### 🥉 Только для экстренных случаев: Развёртывание локального Skill-Runner в один клик

Это подходит только для пользователей, которые **не знают, как устанавливать и настраивать инструменты агентов, и не могут использовать Docker**. Развёртывание в один клик запускается и останавливается вместе с плагином — закрытие Zotero завершает все задачи, и нет фонового выполнения. Если вы способны устанавливать агентов или использовать Docker, пожалуйста, отдайте предпочтение двум вышеуказанным вариантам.

→ [Развёртывание и настройка Skill-Runner](./skill-runner)

### Generic HTTP

Используется для вызова конкретных HTTP API (таких как сервис разбора документов MinerU), которые не связаны с выполнением моделей ИИ. Настраивается по мере необходимости.

→ [Конфигурация бэкенда Generic HTTP](./generic-http)

## Сравнение типов бэкендов

| Тип | Протокол | Режим выполнения | Рекомендация | Сценарий использования |
|------|----------|---------------|----------------|----------|
| **Бэкенд ACP** | Agent Client Protocol | Локальный подпроцесс | 🥇 Первый выбор | У вас есть инструмент агента ACP, нулевая нагрузка по настройке |
| **Skill-Runner (Docker)** | HTTP API | Постоянный сервис | 🥈 Рекомендуется | Нужно постоянное фоновое выполнение, общий доступ в локальной сети |
| **Skill-Runner (Один клик)** | HTTP API | Запускается/останавливается с плагином | 🥉 Экстренный случай | Вообще не можете устанавливать агентов / Docker |
| **Generic HTTP** | HTTP | Удалённый сервис | По мере необходимости | Вызов конкретных HTTP API (например, MinerU) |

Все бэкенды настраиваются через **[Инструменты → Менеджер бэкендов](backend-manager)**.
