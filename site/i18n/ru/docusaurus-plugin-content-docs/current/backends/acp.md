# Конфигурация бэкенда ACP

## Что такое ACP?

ACP (Agent Client Protocol) — это протокол для взаимодействия с бэкендами агентов. Zotero Agents взаимодействует с локально запущенными процессами агентов (такими как Codex, Claude Code, OpenCode и т.д.) через протокол ACP для обеспечения диалогов и выполнения навыков.

Бэкенд ACP — это **рекомендуемый** способ настройки — если у вас установлен любой совместимый с ACP инструмент агента на вашем компьютере, вы можете использовать его напрямую без дополнительной настройки.

## Почему сначала ACP?

- **Нулевая нагрузка по настройке**: Не нужно развёртывать дополнительные сервисы; используйте инструменты агентов, уже имеющиеся на вашем компьютере
- **Автоматическое управление процессами**: Плагин указывает команду запуска в конфигурации и автоматически управляет жизненным циклом процесса агента
- **Поддержка нескольких агентов**: Настройте несколько различных бэкендов агентов одновременно и переключайтесь между ними по мере необходимости
- **Изоляция конфигурации**: Некоторые агенты (такие как OpenCode и Codex) поддерживают изоляцию каталогов конфигурации и каталогов постоянных сессий через переменные окружения

## Шаги настройки

1. Убедитесь, что у вас установлен хотя бы один совместимый с ACP инструмент агента CLI на вашем компьютере
2. Откройте **Инструменты → [Менеджер бэкендов](backend-manager)**
3. Переключитесь на вкладку **ACP**
4. Выберите ваш инструмент агента из выпадающего списка **Добавить из пресета** или нажмите **Добавить ACP** для ручной настройки
5. Заполните следующие поля:
   - **Отображаемое имя**: Понятное имя (например, "Мой OpenCode")
   - **Команда**: Команда для запуска бэкенда ACP (пресеты заполняют автоматически, но вы также можете изменить вручную)
   - **Аргументы**: Дополнительные аргументы для команды (необязательно)
   - **Переменные окружения**: Дополнительные переменные окружения (необязательно, используются для изоляции конфигурации и т.д.)
6. Нажмите **Сохранить** в правом нижнем углу

### Проверка подключения

После сохранения плагин автоматически обнаруживает возможности бэкенда:
- Проверяет существование команды
- Подключается и инициализируется
- Получает список доступных моделей и режимов
- Вычисляет отпечаток конфигурации для обнаружения последующих изменений

Если обнаружение не удалось, проверьте, что CLI агента установлен правильно и формат команды верен.

## Поддерживаемые пресеты агентов

Плагин предоставляет несколько встроенных пресетов, которые вы можете выбрать напрямую из выпадающего списка **Добавить из пресета**:

| Пресет | Команда | Описание |
|--------|---------|-------------|
| **Codex** | `npx codex acp` | Официальный агент для написания кода от OpenAI |
| **Claude Code** | `npx @anthropic-ai/claude-code acp` | Официальный CLI от Anthropic |
| **OpenCode** | `npx opencode-ai@latest acp` | Универсальный фреймворк агента с поддержкой изоляции переменных окружения |
| **Gemini CLI** | `npx @google/gemini-cli acp` | Google Gemini |
| **Hermes** | `npx hermes acp` | Hermes Agent |
| **Qwen Code** | `qwen-code acp` | Qwen Code |

Вы всё ещё можете вручную изменить любое поле после выбора пресета.

## Рекомендации по настройке переменных окружения

Некоторые агенты поддерживают изоляцию конфигурации и постоянные сессии через переменные окружения; просто добавьте их в редакторе переменных окружения:

| Переменная окружения | Агент | Назначение |
|---------------------|-------|---------|
| `OPENCODE_CONFIG` | OpenCode | Указать независимый каталог конфигурации |
| `OPENCODE_SESSION_DIR` | OpenCode | Указать каталог постоянных сессий |
| `CODEX_CONFIG_DIR` | Codex | Указать независимый каталог конфигурации |

## Типы запросов

Бэкенд ACP поддерживает два типа запросов:
- `acp.prompt.v1` — Диалоговое взаимодействие (ACP Чат)
- `acp.skill.run.v1` — Выполнение навыков (Навыки ACP)

Один и тот же бэкенд ACP может использоваться как для диалогов, так и для запусков навыков одновременно.

## Управление сессиями

- Каждый бэкенд может иметь несколько сессий (диалогов), которые постоянно хранятся в базе данных плагина
- Различные бэкенды ACP могут работать одновременно, не мешая друг другу
- Сессиями можно управлять в [ACP Чате](../sidebar/acp-chat)

## Следующие шаги

После завершения настройки вы можете:
- Общаться с бэкендом в [Боковой панели ACP Чата](../sidebar/acp-chat)
- Просматривать запуски навыков ACP в [Панели мониторинга](../dashboard)
- Использовать бэкенд ACP для выполнения задач в [Списке Workflow](../workflows/)
