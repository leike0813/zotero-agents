# MCP-сервер

## Обзор

MCP (Model Context Protocol) Сервер — это встроенный сервис протокола, который предоставляет вашу библиотеку Zotero и возможности Synthesis как 40+ инструментов MCP. Клиенты, совместимые с MCP (Claude Desktop, Cursor, расширения VS Code и т.д.), могут напрямую обращаться к данным Zotero.

MCP-сервер разделяет базовый реестр возможностей Host Bridge, но следует спецификации протокола MCP (Streamable HTTP транспорт, JSON-RPC 2.0).

## Конфигурация

Zotero → Настройки → Zotero Agents → Host Bridge → **Включить MCP-сервер**

Один флажок переключает сервер вкл/выкл. Включено по умолчанию.

### Ненастраиваемые значения по умолчанию

| Настройка | Значение | Причина |
|---------|-------|--------|
| Адрес прослушивания | `127.0.0.1` | Безопасность: только loopback |
| Проверка источника | Строгая | Только `127.0.0.1`, `localhost`, `[::1]` |
| Ограничение размера запроса | 1 MB | Защита памяти |
| Защита записи | Включена | Все операции записи требуют подтверждения |

## Безопасность

- **Аутентификация Bearer Token**: использует тот же сессионный/мастер-токен, что и Host Bridge
- **Только loopback**: удалённый доступ невозможен
- **Проверка источника**: запросы с других источников отклоняются (403)
- **Ограничение 1 MB**: слишком большие тела отклоняются с кодом 413
- **Однопоточная очередь**: 1 выполняющийся + 8 ожидающих, тайм-аут выполнения 45с, тайм-аут очереди 30с
- **Прерыватели цепи**: 3 сбоя за 5 минут → инструмент приостановлен на 60с

## Подключение клиентов MCP

### Конечная точка

```
http://127.0.0.1:<port>/mcp
```

Порт назначается автоматически (диапазон 26370-26569). Проверьте конечную точку Host Bridge в настройках для фактического порта.

### Пример конфигурации Claude Desktop

```json
{
  "mcpServers": {
    "zotero-skills": {
      "type": "http",
      "url": "http://127.0.0.1:26370/mcp",
      "headers": {
        "Authorization": "Bearer <your-token>"
      }
    }
  }
}
```

Получите токен из Настройки → Host Bridge → **Копировать мастер-токен**.

### Детали протокола

- Транспорт: Streamable HTTP (`POST /mcp`)
- Версия: `2025-06-18`
- Идентичность сервера: `zotero-skills` / `"Zotero Agents Context Broker"` v0.4.0
- `GET /mcp` → 405 (принимается только POST)
- Запросы без `id` → рассматриваются как уведомления (без ответа)
- `id: null` → явно недействительно

## Защита записи

Инструменты записи следуют той же модели подтверждения, что и Host Bridge:

```
Клиент MCP вызывает инструмент записи
  │
  ├── Bearer Token проверен
  ├── Область инструмента извлечена
  ├── Проверка подтверждения:
  │     ├── Инструмент только для чтения → выполнить немедленно
  │     ├── Предварительно одобренная запись → выполнить немедленно
  │     └── Требуется подтверждение → в очередь в UI Zotero
  └── Выполнить / Отклонить
```

Очередь: максимум 50 ожидающих подтверждений; >10 отклонённых записей за 5 минут → прерыватель цепи (отключён на 30с).
