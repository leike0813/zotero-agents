# Настройки

Настройки Zotero Agents находятся в **Zotero → Настройки → Zotero Agents** (Windows/Linux) или **Zotero → Параметры → Zotero Agents** (macOS).

## Настройки Workflow

### Каталог Workflow

- **Путь**: Пользовательский каталог для хранения Workflow
- **Расположение по умолчанию**: `<Данные Zotero>/zotero-agents/data/workflows`
- **Сканировать Workflow**: Нажмите кнопку для повторного сканирования каталога и загрузки всех Workflow

### Каталог навыков

- **Путь**: Пользовательский каталог для хранения пакетов навыков
- **Сканировать**: Нажмите кнопку для сканирования каталога и загрузки навыков

### Официальные пакеты Workflow

Официальные Workflow распространяются через отдельные пакеты контента, отделённые от самого плагина.

![Страница настроек Workflow](/img/docs/preferences_workflow.png)

| Настройка | Тип | Описание |
|---------|------|-------------|
| **Установить официальные пакеты Workflow** | кнопка | Загрузить и установить последний официальный пакет с GitHub / Gitee |
| **Проверить обновления** | кнопка | Проверить наличие новой версии удалённо |
| **Статус** | текст | Отображает текущую установленную версию пакета и информацию о канале |

![Содержимое официального пакета Workflow](/img/docs/preferences_official-workflow-contents.png)

#### Каналы обновлений

Вы можете выбрать один из трёх каналов обновлений:

| Канал | Описание |
|---------|-------------|
| **stable** | Стабильный выпуск (рекомендуется) |
| **beta** | Бета-выпуск, включает будущие функции |
| **dev** | Разработческий выпуск, включает последние экспериментальные изменения |

После переключения каналов нажмите **Проверить обновления** для получения последнего пакета для этого канала.

### Настройки выполнения

- **Включить обратную связь по запуску навыков**: Если включено, запуски навыков могут записывать побочные файлы обратной связи Markdown, которые собираются панелью обратной связи по навыкам Панели мониторинга

## Host Bridge

Встроенный HTTP-сервис для внешних ИИ-инструментов и CLI-доступа к библиотеке Zotero. Подробнее см. [Host Bridge](backends/host-bridge).

| Настройка | Тип | Описание |
|---------|------|-------------|
| **Включить MCP-сервер** | boolean | Также предоставить интерфейс протокола MCP |
| **Отключить подтверждение записи** | boolean | Опасно: обойти все подтверждения записи |
| **Включить доступ по локальной сети** | boolean | Разрешить доступ по локальной сети |
| **Фиксированный порт** | boolean | Использовать фиксированный порт вместо случайного |
| **Номер порта** | number | Значение фиксированного порта (по умолчанию 26570) |
| **LAN IP** | string | Вручную указать объявляемый IP (оставьте пустым для автоопределения) |

![Страница настроек Host Bridge](/img/docs/preferences_host-bridge.png)

Кнопки действий:

- **Запустить/Показать конечную точку**: Запустить сервис и отобразить URL конечной точки
- **Сменить токен**: Сменить сессионный токен
- **Создать/Сменить мастер-токен**: Сгенерировать постоянный токен
- **Копировать мастер-токен**: Копировать в буфер обмена
- **Копировать профиль удалённого CLI**: Получить конфигурацию удалённого подключения
- **Установить CLI**: Установка `zotero-bridge` в один клик

![Расширенная область опасных действий Host Bridge](/img/docs/preferences_host-bridge_expand.png)

## Локальный бэкенд SkillRunner

> ⚠️ Этот режим подходит только для пользователей, которые совершенно не знакомы с установкой инструментов агентов и не могут использовать Docker. Если у вас уже есть агент ACP или вы можете использовать Docker, пожалуйста, отдайте предпочтение [бэкенду ACP](backends/acp) или [Skill-Runner, развёрнутому в Docker](backends/skill-runner#рекомендуется-постоянное-развёртывание-в-docker).

Локальный Skill-Runner запускается и останавливается вместе с плагином — закрытие Zotero завершает все задачи. Функции управления выполнением:

| Функция | Описание |
|---------|-------------|
| **Развёртывание в один клик** | Загрузить и установить последнюю версию среды выполнения Skill-Runner |
| **Запустить** | Запустить локальный процесс Skill-Runner |
| **Остановить** | Остановить работающий локальный Skill-Runner |
| **Удалить** | Удалить установленные файлы среды выполнения |
| **Открыть UI управления** | Открыть интерфейс управления бэкендом в плагине |
| **Открыть папку навыков** | Открыть каталог, где хранятся файлы навыков |
| **Обновить кэш моделей** | Обновить кэш списка моделей бэкенда |
| **Открыть консоль отладки** | Просмотр вывода логов бэкенда |

![Страница настроек локального бэкенда SkillRunner](/img/docs/preferences_skillrunner-local-backend.png)

## Менеджер бэкендов

Управление всеми профилями бэкендов:

- Группировка по провайдеру (SkillRunner, ACP, Generic HTTP)
- Добавление/редактирование/удаление бэкендов
- Каждый бэкенд можно настроить с помощью: ID, Base URL, Bearer Token, Timeout

## Синхронизация WebDAV

Решение для синхронизации между устройствами для Synthesis Workbench, заменяющее устаревшую синхронизацию Git. Подробнее см. [Синхронизация WebDAV](synthesis/webdav-sync).

| Настройка | Тип | По умолчанию | Описание |
|---------|------|---------|-------------|
| **Включить синхронизацию WebDAV** | boolean | `false` | Главный переключатель |
| **Base URL** | string | `""` | Адрес сервера WebDAV |
| **Удалённый путь** | string | `"zotero-agents"` | Путь удалённого каталога |
| **Имя пользователя** | string | `""` | Имя пользователя WebDAV |
| **Пароль/Токен** | зашифрованный | `""` | Пароль или токен приложения (шифрование AES-256-GCM) |
| **Авто синхронизация** | boolean | `false` | Автоматически запускать синхронизацию после каждого изменения |
| **Авто повтор** | boolean | `false` | Автоматически повторять при сбое |

Кнопки действий: Сохранить настройки, Сохранить учётные данные, Проверить подключение.

![Страница настроек синхронизации WebDAV](/img/docs/preferences_WebDAV-sync.png)

## Данные выполнения

Отображает корневой каталог постоянных данных, использование во время выполнения и диагностику целостности:

- **Корень постоянных данных**: `<Данные Zotero>/zotero-agents/data/`
- **Каноническое хранилище Synthesis**: Локальный SQLite + постоянные пакеты
- **Размеры каталогов**: data/, cache/, logs/, tmp/ и т.д.
- **Панель диагностики**: Обнаружение проблем файловой системы (например, неочищенные файлы WAL)

Примечание: Каноническое хранилище Synthesis и базы данных состояния предназначены только для диагностики и не могут быть очищены здесь.

![Страница управления данными выполнения и постоянными данными](/img/docs/preferences_storage-and-persistence.png)

## Общие параметры

- **Бэкенд по умолчанию**: Выберите экземпляр бэкенда по умолчанию для использования
- **Автозапуск локального бэкенда**: Автоматически запускать Skill-Runner при запуске Zotero
- **Уровень логирования**: Установить уровень логирования
- **Включить встроенный Markdown-ридер**: Если включено, двойной клик по вложениям `.md` открывает их во встроенном ридере; если выключено, восстанавливается системный открыватель по умолчанию (включено по умолчанию)
