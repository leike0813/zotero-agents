# База знаний концепций

База знаний концепций (Concept KB) — это дополнительный слой знаний в системе Synthesis, обеспечивающий структурированное управление ключевыми концепциями, на которые ссылаются в литературе. Концепции могут накладываться на граф тем и ридер, обогащая контекст синтеза тем.

## Что такое концепция?

В системе Synthesis **концепция** — это термин или сущность с самостоятельным значением в исследовательской области. В отличие от плоской классификации тегов, концепции могут иметь многослойную структуру, включая смыслы, псевдонимы и связи.

### Четырёхслойная структура концепций

```
Концепция                — например, «Transformer»
  └── Смысл              — например, «Transformer (архитектура машинного обучения)»
       ├── Псевдоним     — например, «Модель Transformer», «Сеть Transformer»
       └── Отношение     — broader_than «Механизм внимания»
```

### Типы концепций

| Тип | Описание | Примеры |
|------|-------------|----------|
| `method` | Исследовательские методы | Глубокое обучение, обучение с подкреплением |
| `model` | Модели или архитектуры | Transformer, ResNet |
| `dataset` | Наборы данных | ImageNet, COCO |
| `metric` | Метрики оценки | BLEU, F1-мера |
| `field` | Исследовательские области | Компьютерное зрение, обработка естественного языка |
| `task` | Задачи | Классификация изображений, машинный перевод |
| `tool` | Инструменты | PyTorch, TensorFlow |

## Функции поверхности концепций

### Список концепций

На странице Synthesis Workbench → Концепции вы можете просмотреть все проиндексированные концепции:

- **Фильтрация**: По типу (method / model / dataset и т. д.), статусу или связанным темам
- **Поиск**: Поиск концепций по названию
- **Переключение вида**: Компактный / комфортный режим отображения

![Synthesis Concepts Page](/img/docs/synthesis/concepts.png)

### Детали концепции

После выбора концепции вы можете просмотреть и отредактировать:

| Информация | Описание |
|-------------|-------------|
| **Идентификация** | Идентификатор концепции, название, тип |
| **Статус** | active / deprecated / pending |
| **Определение** | Описательное определение концепции |
| **Смыслы** | Конкретные значения концепции в различных контекстах |
| **Псевдонимы** | Альтернативные названия одной концепции |
| **Отношения** | Связи с другими концепциями (broader / narrower / related) |
| **Связанные темы** | Темы, ссылающиеся на эту концепцию |

### Управление смыслами

Одна и та же концепция может иметь различные значения в разных дисциплинах. Механизм смыслов позволяет:

- Добавлять несколько смыслов к концепции, каждый с собственным определением
- Аннотировать контекст использования или область для каждого смысла
- Связывать конкретные смыслы со статьями или темами

### Управление псевдонимами

- Запись различных соглашений об именовании для одной концепции (например, полное название, аббревиатура, альтернативные термины)
- Псевдонимы используются для сопоставления цитирований и идентификации концепций

### Функции наложения

Информация о концепциях может накладываться на другие поверхности:

- **Наложение на граф тем**: Отображение концепций, связанных с темой, в графе тем
- **Наложение на ридер**: Отображение карточек концепций на странице деталей темы

## Рецензирование

Предложения изменений в базе знаний концепций (новые концепции, новые смыслы, новые отношения) отображаются на вкладке рецензирования концепций в [центре рецензирования](review). Вы можете рецензировать и решать, принимать ли эти предложения.

## Связь с тегами

Концепции и теги — два взаимодополняющих подхода к организации знаний:

| Измерение | Теги | Концепции |
|-----------|------|----------|
| Структура | Плоская, аспект:значение | Многослойная (смыслы + псевдонимы + отношения) |
| Назначение | Классификация и фильтрация литературы | Управление знаниями и анализ ассоциаций |
| Источник | Контролируемый словарь + ИИ-вывод | Автоматическое извлечение из литературы + управление пользователем |
| Охват | Охватывает всю литературу | Глубокое охватывание выбранных ключевых терминов |
