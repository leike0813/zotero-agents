# Manuscript Literature Framing

## Назначение

Помощь в написании разделов Introduction и Related Work академической статьи. Через интерактивный диалог уточните позиционирование статьи, соберите релевантную литературу, проанализируйте фреймворки написания и сгенерируйте черновики LaTeX.

## Сценарии использования

- Составление черновика статьи и необходимость организации фреймворка литературы
- Определение позиционирования и инноваций статьи
- Генерация черновиков LaTeX для разделов Introduction и Related Work

## Ограничения ввода

| Тип ограничения | Описание |
|---------|------|
| Единица ввода | workflow (не нужно выбирать элементы) |
| Способ запуска | Запуск напрямую из Панели мониторинга |

## Поток выполнения

Этот Workflow выполняется интерактивно, проходя через следующие этапы:

```
1. Подтверждение информации о статье
   └── Подтверждение названия статьи и области исследования
       └── Уточнение целевого журнала/места и стиля написания

2. Сбор материалов
   └── Извлечение релевантной литературы из библиотеки Zotero
       └── Получение метаданных литературы и информации о цитировании

3. Многоперспективный анализ фреймворка
   └── Анализ позиционирования статьи в области
       └── Идентификация доступных углов написания и повествовательных нитей

4. План написания
   └── Генерация плана структуры Introduction
       └── Генерация плана организации Related Work

5. Генерация черновика
   └── Вывод черновика LaTeX Introduction
       └── Вывод черновика LaTeX Related Work
       └── Включение отображения цитирований и инвентаря доказательств
```

### Детали взаимодействия

- Каждый этап требует подтверждения от пользователя перед продолжением
- Пользователь может корректировать направление во время диалога
- Прогресс можно отслеживать в Панели мониторинга

## Оценочная продолжительность

Зависит от количества оборотов диалога и размера библиотеки литературы. Этап анализа ИИ занимает приблизительно 5-10 минут, плюс время подтверждения от пользователя для каждого этапа.

## Выходы

После завершения выполнения артефакты могут быть записаны в Zotero (как заметки) через хук Apply Result или загружены:

| Артефакт | Формат | Описание |
|------|------|------|
| `introduction.tex` | LaTeX | Черновик Introduction |
| `related-work.tex` | LaTeX | Черновик Related Work |
| `framing-analysis.json` | JSON | Многоперспективный анализ фреймворка |
| `writing-plan.json` | JSON | План написания |
| `evidence-inventory.json` | JSON | Инвентарь доказательств/цитирований |
| `citation-map.json` | JSON | Отношения отображения цитирований |
| `intent-brief.json` | JSON | Сводка позиционирования статьи |

:::tip Доступ к артефактам
Сгенерированные черновики LaTeX и другие артефакты могут быть найдены в **области артефактов Панели мониторинга**. Вы можете напрямую поместить артефакты в ваш манускрипт LaTeX или экспортировать их для дальнейшей обработки.
:::

## Параметры

| Параметр | Тип | Описание | По умолчанию |
|------|------|------|--------|
| `paperTitle` | string | Название статьи | — |
| `language` | string | Язык вывода | `auto` |
| `targetVenue` | string | Целевой журнал/место (необязательно) | Пусто |
| `articleType` | string | Тип статьи | `original research` |
| `stylePreference` | string | Предпочтение стиля написания (необязательно) | Пусто |

### Примеры стиля написания

- `concise`: Лаконичный стиль
- `IEEE-like`: Стиль IEEE
- `Nature-like`: Стиль Nature
- `Chinese draft`: Китайский черновик

## Зависимости

- **Бэкенд**: Бэкенд ACP
- **Библиотека Zotero**: Требуются связанные элементы статей в библиотеке

:::tip Рекомендуемый Workflow
Для лучших результатов рекомендуется выполнить следующую подготовку перед запуском этого Workflow:
1. Соберите и примите достаточное количество связанных статей
2. Запустите [Анализ литературы](literature-analysis) + [Tag Regulator](tag-regulator) на всех статьях
3. Запустите Расширенное сопоставление в Synthesis Workbench и обработайте элементы подтверждения
4. Создайте несколько связанных [Topic Syntheses](topic-synthesis)
:::

## Рекомендации по модели

🟡 Рекомендуются модели с **длинным контекстом**. Написание Introduction и Related Work требует интеграции дайджестов, анализа цитирований и результатов Topic Synthesis из большого количества статей, что предъявляет высокие требования к окну контекста.

## Связанные Workflow

- [Анализ литературы](literature-analysis) — Создание структурированной основы знаний для статей
- [Topic Synthesis](topic-synthesis) — Сначала создайте синтез тем, затем напишите статью на основе результатов анализа
