# Вызов и настройка Workflow

## Способы вызова

![Кнопка панели инструментов Запустить Workflow](/img/icon_play.png)

### Через контекстное меню

1. Выберите один или несколько элементов в списке элементов Zotero
2. Щёлкните правой кнопкой мыши и выберите подменю **Zotero Agents**
3. Выберите Workflow из списка
4. Если появится диалог конфигурации, заполните параметры и нажмите Запустить

### Через Панель мониторинга

1. Откройте **Панель мониторинга** (кнопка на панели инструментов или меню)
2. Найдите целевой Workflow в списке Workflow на главной странице
3. Нажмите кнопку **Запустить**
4. Если появится диалог конфигурации, заполните параметры и отправьте

## Диалог настроек Workflow

Перед запуском Workflow может появиться диалог настроек со следующими параметрами конфигурации:

### Настройки параметров

Отображает все настраиваемые параметры, объявленные Workflow, которые различаются в зависимости от определения Workflow.

### Опции провайдера

| Опция | Описание |
|------|------|
| Выбор бэкенда | Выберите экземпляр бэкенда для выполнения этого Workflow |
| Выбор модели | Модель ИИ для использования (предоставляется бэкендом) |
| Настройки режима | Конфигурация режима выполнения |
| Усилие рассуждения | Уровень усилия рассуждения (если поддерживается бэкендом) |

### Режимы выполнения

| Режим | Описание |
|------|------|
| `auto` | Автоматическое выполнение, не требуется вмешательство пользователя |
| `sync` | Синхронное выполнение, ожидание результатов |
| `async` | Асинхронное выполнение, работает в фоновом режиме |

### Режимы SkillRunner

Для бэкендов Skill-Runner:

| Режим | Описание |
|------|------|
| `auto` | Неинтерактивное выполнение, подходит для навыков, не требующих ввода от пользователя |
| `interactive` | Интерактивное выполнение, может потребоваться ввод от пользователя во время выполнения |

## Выполнение и мониторинг

- После отправки задачи вы можете просматривать прогресс выполнения в Панели мониторинга
- Обновления статуса в реальном времени (в очереди → выполняется → успешно/ошибка/отменено)
- Для интерактивных Workflow вы можете отвечать на задачи, ожидающие ввода, в боковой панели
- После завершения выполнения результаты применяются к Zotero через скрипты хуков

## Примечания

- При первом запуске Workflow может потребоваться настройка бэкенда
- Некоторые Workflow могут иметь специальные требования к вводу (например, должны быть выбраны вложения)
- Интерактивные Workflow требуют, чтобы Zotero оставался запущенным для обработки ввода от пользователя
