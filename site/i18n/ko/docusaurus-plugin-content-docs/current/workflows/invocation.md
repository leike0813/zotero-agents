# Workflow 호출 및 설정

## 호출 방법

![Workflow 실행 도구 모음 단추](/img/icon_play.png)

### 컨텍스트 메뉴를 통한 호출

1. Zotero 항목 목록에서 하나 이상의 항목을 선택합니다
2. 마우스 오른쪽 버튼을 클릭하고 **Zotero Agents** 하위 메뉴를 선택합니다
3. 목록에서 Workflow를 선택합니다
4. 설정 대화 상자가 나타나면 파라미터를 입력하고 실행을 클릭합니다

### 대시보드를 통한 호출

1. **대시보드**를 엽니다 (도구 모음 단추 또는 메뉴)
2. 홈 페이지의 Workflow 목록에서 대상 Workflow를 찾습니다
3. **실행** 단추를 클릭합니다
4. 설정 대화 상자가 나타나면 파라미터를 입력하고 제출합니다

## Workflow 설정 대화 상자

Workflow를 실행하기 전에 다음 설정 옵션을 포함하는 설정 대화 상자가 나타날 수 있습니다:

### 파라미터 설정

Workflow에서 선언한 모든 설정 가능한 파라미터를 표시하며, Workflow 정의에 따라 달라집니다.

### 제공자 옵션

| 옵션 | 설명 |
|------|------|
| 백엔드 선택 | 이 Workflow를 실행할 백엔드 인스턴스를 선택합니다 |
| 모델 선택 | 사용할 AI 모델 (백엔드에서 제공) |
| 모드 설정 | 실행 모드 설정 |
| 추론 강도 | 추론 강도 수준 (백엔드에서 지원하는 경우) |

### 실행 모드

| 모드 | 설명 |
|------|------|
| `auto` | 자동 실행, 사용자 개입 불필요 |
| `sync` | 동기 실행, 결과를 기다립니다 |
| `async` | 비동기 실행, 백그라운드에서 실행됩니다 |

### SkillRunner 모드

Skill-Runner 백엔드의 경우:

| 모드 | 설명 |
|------|------|
| `auto` | 비인터랙티브 실행, 사용자 입력이 필요 없는 스킬에 적합 |
| `interactive` | 인터랙티브 실행, 실행 중 사용자 입력이 필요할 수 있음 |

## 실행 및 모니터링

- 작업이 제출되면 대시보드에서 실행 진행 상황을 확인할 수 있습니다
- 실시간 상태 업데이트 (대기 중 → 실행 중 → 성공/실패/취소)
- 인터랙티브 Workflow의 경우 사이드바에서 입력을 기다리는 작업에 답변할 수 있습니다
- 실행이 완료되면 훅 스크립트를 통해 결과가 Zotero에 적용됩니다

## 참고 사항

- Workflow를 처음 실행하려면 백엔드 설정이 필요할 수 있습니다
- 일부 Workflow는 특정 입력 요구 사항이 있을 수 있습니다 (예: 첨부파일을 선택해야 함)
- 인터랙티브 Workflow는 사용자 입력을 처리하기 위해 Zotero가 실행 상태로 유지되어야 합니다
