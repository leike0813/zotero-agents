# 인덱스 & 인용 그래프

## 인덱스 서피스

Synthesis Workbench → Index 페이지에서 Canonical Reference Index를 관리할 수 있습니다. 인덱스 서피스는 **두 개의 서브 뷰**를 포함합니다:

### Registry 뷰

라이브러리의 모든 추적된 논문 리스트를 표시하며, 각 행은 논문과 그 커버리지 상태를 보여줍니다:

- **논문 정보**: 제목, 저자, 연도
- **커버리지**: Complete / Partial / Missing (세 가지 아티팩트 유형의 커버리지 상태: digest, references, citation analysis)
- **행 확장**: 확장하면 논문의 참고문헌 리스트가 표시되며, 각 인용은 바인딩 상태별로 표시됩니다 (미바인딩 / 후보 / 수락됨 / 거절됨)
- **필터**: 범위 (전체 / 라이브러리 / 인용됨), 커버리지 또는 검색으로 필터링

![Synthesis Index Registry View](/img/docs/synthesis/index.png)

### 표준 참고문헌 뷰

활성 인덱싱 도구가 "Revise Canonical"로 전환될 때 표시됩니다:

- **표준 참고문헌 리스트**: 중복 제거된 표준 참고문헌 레코드
- **검색 및 필터**: 바인딩 상태, 그래프 표시 여부, 리다이렉트 상태 또는 중복 후보 존재 여부로 필터링
- **작업**: 메타데이터 편집, 중복 참고문헌 병합, 리다이렉트 생성, 리뷰 항목 보기

![Synthesis Index Canonical Reference Revision View](/img/docs/synthesis/index_canonical-revision.png)

## Canonical Reference Index

Canonical Reference Index는 Synthesis 시스템의 핵심 인덱스로, 라이브러리의 논문에서 모든 참고문헌의 중복 제거 및 표준화를 수행합니다. Reference Sidecar에서 원시 인용 데이터를 가져오고 ([개요](/synthesis)의 "Reference Sidecar" 섹션 참조), 추출, 표준화 및 매치 바인딩을 통해 인덱스를 형성합니다.

### 기능

- **전체 텍스트 검색**: 모든 표준화된 참고문헌에 대해 검색
- **메타데이터 편집**: 인용 레코드 메타데이터 수정
- **병합**: 중복 참고문헌 레코드 병합 (자동으로 리다이렉트 생성)
- **리다이렉트**: 하나의 참고문헌을 다른 표준 레코드로 지정
- **리뷰**: 인용 매칭의 품질 리뷰 항목 보기
- **중복 제거**: 잠재적 중복 참고문헌 발견

### 참고문헌 레코드 유형

| 유형 | 설명 |
|------|-------------|
| **바인딩됨** | Zotero 라이브러리의 항목과 연관됨 |
| **외부** | 현재 Zotero 라이브러리에 없는 알려진 문헌 |
| **미해결** | 참고문헌에서 추출되었지만 아직 식별되지 않음 |

## 참고문헌 매칭 파이프라인

참고문헌 매칭은 논문에서 추출된 참고문헌과 Zotero 라이브러리 항목 간의 연관을 자동으로 설정하는 프로세스입니다. 시스템은 **2단계 모델**을 사용하여 성능과 정확도의 균형을 맞춥니다.

### 2단계 모델

#### 1단계: 경량 Sidecar 새로 고침

정기 작업 중(예: digest 적용 후)에 실행되며, sidecar 상태를 스캔하고 인용 아티팩트 해시를 비교하며 변경된 참고문헌만 처리합니다. **고급 중복 제거나 인덱스 빌딩을 실행하지 않으며**, 경량 표준 할당 및 바인딩만 수행합니다.

- 트리거: Workflow 실행이 완료되고 아티팩트가 기록된 후, 또는 명시적인 새로 고침 작업을 통해
- 범위: 증분 (변경된 참고문헌만)
- 알고리즘: 단순 식별자 매칭 (DOI, arXiv, ISBN)

#### 2단계: 고급 인용 매칭

명시적으로 트리거되는 심층 매칭 작업입니다. 완전한 인용 매치 인덱스를 빌드하고 포괄적인 매칭 및 중복 제거 알고리즘을 실행합니다.

- 트리거: 사용자의 수동 트리거, 정기 유지보수
- 범위: 전체
- 알고리즘: 다중 전략 매칭 + 클러스터링 중복 제거

:::caution 성능 안내
고급 인용 매칭, 인덱스 새로 고침 및 인용 그래프 재구축은 계산 집약적입니다. Zotero는 단일 호스트 프로세스 아키텍처를 사용하므로, 이러한 작업은 실행 중 짧은 UI 끊김을 일으킬 수 있습니다. 잠시만 기다려 주세요. 이 문제는 향후 아키텍처 리팩토링에서 해결될 예정입니다.
:::

### 매칭 전략

| 전략 | 매칭 기준 | 신뢰도 | 설명 |
|----------|-------------|------------|-------------|
| DOI 매칭 | DOI 식별자 | 결정적 | 완전 일치, 자동 수락 |
| arXiv 매칭 | arXiv ID | 결정적 | 완전 일치, 자동 수락 |
| ISBN 매칭 | ISBN 번호 | 결정적 | 완전 일치, 자동 수락 |
| 제목 유사도 | 퍼지 제목 매칭 | 높음 / 중간 / 낮음 | 표준화된 제목과 축약 제목을 사용하여 유사도 계산 |
| 저자 + 연도 | 저자 이름 및 출판 연도 | 중간 / 낮음 | 저자 정규화와 연도 범위를 결합하여 매칭 |

### 신뢰도 수준

| 수준 | 설명 | 권장 작업 |
|-------|-------------|-------------------|
| `deterministic` | 결정적 매치 | 자동 수락 |
| `high` | 고신뢰도 | 수락 가능 |
| `medium` | 중신뢰도 | 리뷰 권장 |
| `low` | 저신뢰도 | 리뷰 필요 |
| `review` | 인간 판단 필요 | 반드시 리뷰 필요 |

### 클러스터링 중복 제거

고급 매칭 단계에서는 표준 참고문헌에 대해 클러스터링 중복 제거를 수행합니다. 알고리즘 과정:

1. 각 표준 참고문헌에 대한 중복 제거 레코드 빌드 (적격성 필터링 및 서지 노이즈 분석 포함)
2. 쌍별 비교를 통해 클러스터 엣지 생성 (식별자 완전 일치, 제목 표준 일치, 퍼지 제목 매칭 등)
3. 엣지가 클러스터 및 서브클러스터로 집계됨
4. 중복 제거를 위한 자동 리다이렉트 또는 리뷰 제안 생성

안전 제약: 저신뢰도 매치(예: `contained_extension_risk`)는 자동 리다이렉트를 트리거하지 않으며 사용자 리뷰가 필요합니다.

### 리뷰 서피스

[리뷰 허브](review)에서 인용 매치 제안을 보고 처리할 수 있으며, 하나씩 수락 또는 거절할 수 있습니다.

## 인용 그래프

인용 그래프는 라이브러리의 논문과 그 참고문헌을 네트워크 그래프로 시각화합니다. 그래프 데이터는 SQLite 프로젝션으로 빌드되며 일정 수준의 데이터 오래됨을 허용합니다 (실시간 미러가 아님).

![Synthesis Citation Graph](/img/docs/synthesis/citation-graph.png)

### 노드 유형

| 노드 | 색상 | 설명 |
|------|-------|-------------|
| `library_paper` | 파랑 | 이미 Zotero 라이브러리에 있는 논문 |
| `external_reference` | 초록 | 라이브러리에 없는 알려진 참고문헌 |
| `unresolved_reference` | 회색 | 추출되었지만 식별되지 않은 참고문헌 |

### 엣지 정보

각 인용 엣지에는 다음이 포함됩니다:

- **mention_count**: 인용된 횟수
- **primary_role**: 주요 인용 역할 (예: 배경, 비교, 지지, 대조)
- **aux_roles**: 보조 역할 리스트
- **role_evidence**: 역할 결정의 근거

### 그래프 지표

인용 그래프는 핵심 논문과 영향력 있는 작업을 식별하는 데 도움이 되는 다양한 지표를 계산할 수 있습니다:

| 지표 | 설명 |
|--------|-------------|
| **인용 수** | 논문이 인용된 총 횟수 |
| **PageRank** | 그래프 구조 기반 노드 중요도 점수 |
| **기초 점수** | 해당 분야에서 기초 작업으로 사용되는 정도 |
| **최전선 점수** | 최전선 작업을 나타내는 정도 |

### 시각화 레이아웃

| 레이아웃 | 설명 | 사용 사례 |
|--------|-------------|----------|
| **Force (힘 방향)** | d3-force 레이아웃 | 전체 구조 탐색 |
| **Radial** | 선택된 노드 주변으로 확장 | 한 논문의 인용 네트워크 분석 |
| **Components** | 연결된 컴포넌트별 그룹화 | 독립적인 인용 클러스터 발견 |

### 인터랙티브 작업

- **확대/축소 / 패닝**: 그래프를 자유롭게 탐색
- **호버**: 노드 레이블 및 기본 정보 보기
- **노드 클릭**: Zotero에서 해당 논문 항목 열기
- **필터**: 역할, 토픽 또는 노드 유형별로 표시되는 인용 필터링
- **저신호 인용 토글**: 낮은 인용 수의 엣지 표시/숨김
- **깊이 슬라이더**: 인용 네트워크의 확장 깊이 제어

### 토픽 필터링

토픽별로 인용 그래프를 필터링하여 특정 토픽과 관련된 논문 및 인용 관계만 표시할 수 있습니다. 토픽 범위는 그래프에서 서로 다른 색상과 그룹으로 표시됩니다.
