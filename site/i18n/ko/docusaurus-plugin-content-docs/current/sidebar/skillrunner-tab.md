# SkillRunner 탭

SkillRunner 탭은 Skill-Runner 백엔드를 통해 실행된 실행을 확인하고 상호작용하는 데 사용됩니다. 일회성 스킬 실행에 중점을 두는 ACP Skills와 달리, SkillRunner 탭은 인터랙티브 세션 관리에 중점을 둡니다.

## 인터페이스 개요

![SkillRunner Panel](/img/docs/sidebar/skillrunner-tab.png)

```
┌─────────────────────────────────────┐
│  배너: 제목 / requestId / 상태       │
├─────────────────────────────────────┤
│  ← Task Drawer  │  메인 콘텐츠 영역 │  Details → │
│               │  Transcript View            │
│  실행 중       │  Plan Component             │
│  └─ backend1  │  Prompt Component           │
│     └─ task A │  Reply Area                 │
│  완료          │                             │
│  └─ backend1  │                             │
│     └─ task B │                             │
└─────────────────────────────────────┘
```

## 배너

배너에는 현재 선택된 작업에 대한 정보가 표시됩니다:

- **제목**: 작업 이름 또는 스킬 식별자
- **Request ID**: 작업의 고유 요청 식별자
- **상태**: 실행 상태 (실행 중 / waiting_user / waiting_auth / 완료 / 실패)
- **백엔드**: 백엔드 정보
- **엔진**: 사용 중인 엔진 (예: gemini, claude 등)
- **모델**: 사용 중인 모델
- **업데이트**: 마지막 업데이트 시간
- **작업 취소 버튼**

## 작업 서랍 (왼쪽)

왼쪽 서랍에는 모든 SkillRunner 작업이 표시되며, 실행 중과 완료 그룹으로 나뉩니다. 각 작업 항목에는 요약 정보, 상태 표시등 및 보관 작업이 표시됩니다. 항목을 클릭하면 해당 작업의 상세 보기로 전환됩니다.

## 메인 콘텐츠 영역

### 트랜스크립트 뷰

SkillRunner 트랜스크립트 뷰는 지속적인 추론을 지능적으로 처리하는 **Thinking chat 모델**을 사용합니다:

- **Thinking 블록**: AI의 추론 과정이 별도의 thinking 블록으로 표시됩니다
- **도구 호출**: 도구 이름, 입력 요약 및 실행 상태를 표시합니다
- **메시지**: 어시스턴트 및 사용자 대화 메시지
- **수정**: 출력 버전 변경 기록

**Plain / Bubble** 표시 모드도 지원합니다.

### 인증 워크플로우

SkillRunner 탭은 인증 워크플로우를 지원하여, 패널을 벗어나지 않고 백엔드 인증을 완료할 수 있습니다:

**인증 트리거:**

- 인증이 필요한 스킬을 실행할 때 자동으로 트리거됩니다
- 프롬프트 컴포넌트에 인증 요청이 표시됩니다

**지원되는 인증 방법:**

| 방법 | 설명 | 사용 사례 |
|------|------|---------|
| **OAuth 프록시** | 브라우저를 통해 OAuth 플로우를 완료합니다 | 권장 방법. OAuth를 지원하는 엔진용 |
| **인증 코드 입력** | 인증 코드 또는 URL을 수동으로 입력합니다 | 엔진이 인증 링크를 생성한 경우 |
| **파일 가져오기** | 자격 증명 파일을 가져옵니다 | 자격 증명 파일이 이미 있는 경우 |
| **Inline TUI** | 패널 내에서 직접 터미널을 실행합니다 | 인터랙티브 로그인이 필요한 경우 |

**인증 플로우 예시 (OAuth):**

1. 실행 중 인증이 필요함을 감지합니다
2. 프롬프트 컴포넌트에 "인증이 필요합니다"와 사용 가능한 인증 방법이 표시됩니다
3. 사용자가 OAuth 프록시를 선택합니다
4. 브라우저에서 OAuth 페이지가 열립니다
5. 사용자가 인증을 완료합니다
6. 실행이 자동으로 재개됩니다

### 프롬프트 컴포넌트

| 상태 | 표시 내용 |
|------|---------|
| `waiting_user` | 사용자 입력 대기 중, 컨텍스트 설명과 빠른 옵션(사용 가능한 경우)이 표시됩니다 |
| `waiting_auth` | 인증 대기 중, 인증 방법 선택 및 입력이 표시됩니다 |
| `running` | 진행 중 표시등 |
| `completed` | 완료 상태 확인 |
| `error` | 오류 정보와 문제 해결 제안 |

### 응답 영역

- **텍스트 입력 박스**: 응답 내용을 입력합니다
- **전송/취소 버튼**

ACP Skills와 달리, SkillRunner 탭의 응답 영역에는 모드/모델/추론 선택기가 없습니다(이들은 백엔드 설정에서 구성됩니다).

## 상세 정보 서랍 (오른쪽)

| 영역 | 내용 |
|------|------|
| **실행 메타데이터** | 제목, requestId, taskKey, 상태, terminal/waiting 플래그 |
| **백엔드 정보** | 백엔드, 엔진, 모델 |
| **업데이트 시간** | 마지막 활동 시간 |
| **상호작용 정보** | 현재 대기 중인 상호작용 정보(있는 경우) |
| **세션 요약** | 과거 세션 요약 |
| **수정 요약** | 출력 버전 변경 기록 |

## 관련 설정

SkillRunner 탭을 사용하려면 먼저 [Skill-Runner 백엔드](../backends/skill-runner)를 설정해야 합니다.
