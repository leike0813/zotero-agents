# ACP Skills

ACP Skills 탭은 ACP 백엔드를 통해 실행된 스킬 실행을 모니터링하고 관리하는 데 사용됩니다. ACP Chat의 지속적인 대화와 달리, ACP Skills는 일회성 또는 주기적으로 실행되는 스킬 작업을 위해 설계되었습니다.

## 인터페이스 개요

ACP Skills 패널은 다음 주요 영역으로 나뉩니다:

![ACP Skills Panel](/img/docs/sidebar/acp-skills.png)

```
┌─────────────────────────────────────┐
│  배너: 작업 제목 / 상태 / 백엔드      │
├─────────────────────────────────────┤
│  ← Run Drawer  │  메인 콘텐츠 영역  │  Details → │
│               │  Transcript View            │
│  실행 중       │  Plan Component             │
│  └─ backend1  │  Prompt Component           │
│     ├─ run A  │  Reply Area                 │
│     └─ run B  │                             │
│  완료          │                             │
│  └─ backend1  │                             │
│     └─ run C  │                             │
└─────────────────────────────────────┘
```

## 배너

배너 영역에는 현재 선택된 실행의 메타 정보와 액션 버튼이 표시됩니다:

- **작업 제목**: 실행의 스킬 이름
- **상태**: 실행 상태 표시등 (실행 중 / 완료 / 실패 / 취소 등)
- **백엔드**: 실행을 수행하는 ACP 백엔드
- **액션 버튼**: 연결/연결 해제, 작업 취소

## 실행 서랍 (왼쪽)

왼쪽 서랍에는 모든 ACP Skill 실행이 트리 구조로 정리되어 있습니다:

### 그룹화

| 그룹 | 설명 |
|------|------|
| **실행 중** | 현재 실행 중인 작업, 백엔드별 그룹화 |
| **완료** | 완료된 작업, 백엔드별 그룹화 |

각 작업 항목에는 요약 정보(스킬 ID, 상태, 시간)가 표시되며, 상태 변경을 표시하는 주의 표시등(LED)이 있습니다. 작업 항목을 클릭하면 해당 실행의 상세 보기로 전환됩니다.

### 보관

완료된 작업은 보관 버튼을 통해 목록에서 제거할 수 있습니다 (보관은 현재 세션에서 숨기기만 하며 실행 기록에는 영향을 주지 않습니다).

## 메인 콘텐츠 영역

### 트랜스크립트 뷰

실행을 선택하면 메인 콘텐츠 영역에 해당 실행의 전체 트랜스크립트가 표시됩니다:

- **메시지**: 어시스턴트 및 사용자 대화 내용
- **도구 호출**: AI가 호출한 도구와 그 결과로, 도구 이름, 입력 요약 및 상태 LED가 표시됩니다
- **사고 과정**: AI의 추론 과정 (사용 가능한 경우)
- **상태 이벤트**: 실행 중 상태 변경

트랜스크립트는 **Plain 모드**(메시지가 역할별로 왼쪽 테두리 색상이 구분되어 표시)와 **Bubble 모드**(메시지가 말풍선 스타일로 표시되며, 연속적인 도구 호출이 자동으로 그룹으로 접힘)를 지원하며, 우상단 버튼으로 전환할 수 있습니다.

### 플랜 컴포넌트

실행에 다단계 계획이 포함된 경우, 플랜 컴포넌트가 현재 진행 상황, 완료된 단계 및 대기 중인 단계를 표시하며, 각 단계에는 상태 아이콘(진행 중/완료/실패)이 있습니다.

### 프롬프트 컴포넌트

프롬프트 컴포넌트는 실행 상태에 따라 다른 인터랙티브 프롬프트를 표시합니다:

| 상태 | 표시 내용 |
|------|---------|
| `waiting_user` | 사용자 응답을 기다리는 프롬프트, 컨텍스트 설명과 빠른 응답 옵션이 표시됩니다 |
| `permission` | 권한 요청 프롬프트, 명령 미리보기와 승인/거부 버튼이 표시됩니다 |
| `disconnected` | 재연결 프롬프트, 클릭하여 연결합니다 |
| `running` | 진행 중 표시등 |
| `completed` | 완료 상태 확인 |
| `error` | 오류 정보와 문제 해결 제안 |

### 응답 영역

하단의 응답 영역에는 다음이 포함됩니다:

- **텍스트 입력 박스**: 응답 내용을 입력합니다
- **모드 선택** (선택 사항): 실행 모드 전환
- **모델 선택** (선택 사항): AI 모델 전환
- **Reasoning Effort** (선택 사항): 추론 강도 레벨
- **전송/취소 버튼**
- **사용량 미터**: 토큰 사용량(사용량/제한)을 나타내는 원형 차트
- **키보드 단축키 힌트**: 응답 전송을 위한 키보드 단축키

응답 초안은 요청별로 저장됩니다 — 실행 간 전환 후 다시 돌아와도 전송되지 않은 내용이 유지됩니다.

## 상세 정보 서랍 (오른쪽)

오른쪽 서랍에는 선택된 실행에 대한 상세 정보가 표시되며, 다음 접을 수 있는 영역이 있습니다:

| 영역 | 내용 |
|------|------|
| **실행 경로** | 워크스페이스 디렉토리, 결과 파일 경로 |
| **Runner 정보** | 백엔드, 에이전트, 모드, 모델, 추론, 스킬, 세션 |
| **검증 정보** | 검증 상태, 수정 횟수, 오류 상세 |
| **런타임 의존성** | 런타임 환경 의존성 목록 |
| **출력 수정** | 출력 수정 이력 |
| **런타임 로그** | 실행 중 로그 항목 |
| **결과 JSON** | 최종 구조화된 출력 (확장 가능) |

## 권한 처리

실행이 Zotero 쓰기 권한 또는 ACP 도구 호출 권한을 필요로 할 때, 프롬프트 컴포넌트에 권한 요청이 표시됩니다:

- **명령 미리보기**: 요청되고 있는 작업을 표시합니다
- **출처 정보**: 누가 요청을 시작했는지 표시합니다
- **액션 버튼**: 승인 / 거부
- 확장하여 전체 요청 상세 정보를 확인할 수 있습니다

## 관련 설정

ACP Skills 패널을 사용하려면 먼저 [ACP 백엔드](../backends/acp)를 설정해야 합니다.
