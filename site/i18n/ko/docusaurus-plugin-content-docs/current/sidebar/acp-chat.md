# ACP Chat 사용법

## 기능

ACP Chat에서는 설정된 ACP 백엔드와 대화할 수 있으며, 대화 컨텍스트는 현재 Zotero에서 보고 있는 항목이나 리더의 논문에서 가져옵니다.

## 사용 사례

- **문헌 Q&A**: 현재 읽고 있는 논문에 대한 질문을 하고, 설명과 요약을 받습니다
- **작성 지원**: 작성 과정에서 제안 사항을 받습니다
- **빠른 조회**: 특정 논문의 핵심 정보를 빠르게 검색합니다
- **배치 처리**: 문헌 목록의 여러 항목에 대해 배치 분석을 수행합니다

## 배너

배너는 패널 상단에 위치하며, 핵심 제어 기능을 제공합니다:

### 백엔드 선택

드롭다운에 설정된 모든 백엔드가 나열되며, 각각 상태 접미사(Connecting/Connected/Disconnected)가 표시됩니다. 백엔드를 전환하면 해당 백엔드의 세션으로 자동 전환됩니다.

### 세션 선택

드롭다운에는 최근 8개의 세션이 표시됩니다(시간순 정렬). 세션을 선택하면 해당 세션으로 전환됩니다. 8개 초과 시 하단에 "Show more..."가 표시되며, 클릭하면 세션 서랍이 열려 전체 목록을 확인할 수 있습니다.

### 연결 제어

- **연결/연결 해제 버튼**: 현재 백엔드의 연결 상태를 수동으로 관리합니다
- **인증 버튼**: 백엔드에서 인증이 필요할 때 표시됩니다
- **새 세션(＋)**: 현재 백엔드에 새 세션을 생성합니다

### 상태 표시등

배너 우측에는 세 개의 상태 표시등이 표시됩니다:

| 표시등 | 설명 |
|-----------|-------------|
| ● Connection | ACP 백엔드와의 연결 상태 (초록=연결됨/회색=연결 해제됨/노란=연결 중) |
| ● MCP | MCP 서비스 가용성 |
| ● Host Bridge | Zotero Host Bridge 연결 상태 (아래 참조) |

### Host Bridge 상태

Host Bridge는 Zotero 플러그인과 백엔드 간의 내부 브리지 채널입니다. 현재 Zotero 컨텍스트(선택된 항목, 리더의 논문, 라이브러리 데이터 등)를 백엔드로 전달하는 역할을 담당하며, 이를 통해 AI가 실제 Zotero 데이터를 기반으로 작업할 수 있습니다.

Host Bridge는 `zotero-bridge` CLI 도구를 통해 통신하며, 플러그인은 백그라운드에서 자동으로 생명주기를 관리합니다.

| 상태 | 의미 |
|--------|---------|
| 초록 ● | Host Bridge가 연결됨; 백엔드가 Zotero 컨텍스트에 접근 가능 |
| 노란 ● | 연결 중 또는 재연결 중 |
| 회색 ● | Host Bridge를 사용할 수 없음(설치되지 않거나 시작되지 않음); 백엔드가 Zotero 컨텍스트를 가져올 수 없음 |
| 숨김 | 현재 Host Bridge가 필요하지 않음 (예: 백엔드가 지원하지 않거나 컨텍스트 기능이 비활성화된 경우) |

Host Bridge를 사용할 수 없을 때에도 ACP Chat은 정상적으로 작동하지만, AI가 현재 보고 있는 논문의 정보를 컨텍스트로 접근할 수 없습니다.

## 세션 서랍 (왼쪽)

왼쪽 서랍에는 모든 과거 세션이 백엔드별로 그룹화되어 표시됩니다. 각 세션 항목에는 제목과 마지막 활동 시간이 표시됩니다.

- **세션 전환**: 목록에서 세션을 클릭하여 로드합니다
- **새 세션**: 서랍 상단 또는 배너에서 조작합니다

## 트랜스크립트 뷰

### 대화 메시지

대화 메시지는 Markdown 렌더링을 지원하며, 다음을 포함합니다:

- **코드 블록**: 구문 강조 및 복사 버튼이 포함된 코드 블록
- **수식**: KaTeX로 렌더링되는 LaTeX 수식
- **목록, 표, 링크** 및 기타 표준 Markdown 요소

### 도구 호출

AI가 도구를 호출하면 트랜스크립트에 도구 호출 항목이 표시됩니다:

- 도구 이름 배지
- 입력 파라미터 요약
- 실행 상태 LED (대기/진행 중/완료/실패)
- Bubble 모드에서는 연속적인 도구 호출이 자동으로 "도구 활동 그룹"으로 접힙니다

### 사고 과정

AI의 추론 과정은 정식 답변과 구별되는 별도의 "Thinking" 블록으로 표시됩니다.

### 표시 모드 전환

우상단의 전환 버튼으로 두 가지 모드 간 전환이 가능합니다:

| 모드 | 설명 |
|------|-------------|
| **Plain** | 메시지가 역할별로 왼쪽 테두리 색상이 구분되어 표시되며, 긴 대화를 탐색하기에 적합 |
| **Bubble** | 메시지가 말풍선 스타일로 표시되며, 연속적인 도구 호출이 자동으로 그룹화되어 읽기에 적합 |

### 플랜 컴포넌트

대화에 다단계 계획이 포함된 경우, 트랜스크립트 상단에 계획 진행 바가 표시되어 완료된 단계, 진행 중인 단계, 대기 중인 단계를 표시합니다.

### 프롬프트 컴포넌트

프롬프트 컴포넌트는 사용자 상호작용이 필요할 때 표시됩니다:

- **권한 요청**: 백엔드가 Zotero 접근 권한을 필요로 할 때, 요청 상세 정보와 승인 버튼을 표시합니다
- **연결 프롬프트**: 연결이 해제되었을 때, 재연결 안내를 표시합니다
- **오류 프롬프트**: 오류 정보와 복구 작업을 표시합니다

## 응답 영역

### 텍스트 입력

- **다중 라인 텍스트 박스**: 긴 텍스트 입력을 지원합니다
- **Enter로 전송**: Enter 키를 눌러 메시지를 전송합니다
- **Shift+Enter로 줄바꿈**: 줄바꿈을 삽입합니다
- **응답 이력**: 위/아래 화살표 키를 눌러 전송한 메시지를 탐색합니다

### 실행 모드

응답 영역 상단에서 다음을 선택할 수 있습니다:

| 옵션 | 설명 | 사용 가능한 값 |
|--------|-------------|-----------------|
| **Mode** | 실행 모드 | 백엔드에서 정의 |
| **Model** | AI 모델 | 백엔드가 지원하는 모델 목록 |
| **Reasoning Effort** | 추론 강도 레벨 | Low/Medium/High (백엔드에서 지원하는 경우) |

### 사용량 미터

응답 영역 우하단에 원형 사용량 미터가 표시됩니다:

- **외부 링**: 제한 대비 현재 세션 토큰 사용량의 백분율
- **텍스트**: `사용량 k / 제한 k`
- 사용량 레벨에 따라 색상이 변경됩니다 (보통 → 경고 → 위험)

### 키보드 단축키 힌트

키보드 단축키 힌트는 입력 박스 내부에 표시됩니다.

## 상세 정보 서랍 (오른쪽)

오른쪽 서랍에는 현재 세션에 대한 상세 정보가 표시됩니다:

| 영역 | 내용 |
|------|---------|
| **세션 정보** | 세션 ID, 생성 시간, 마지막 활동 시간 |
| **백엔드 정보** | 백엔드 유형, 주소, 모델 |
| **워크스페이스 경로** | 세션 워크스페이스 파일 경로 |
| **진단** | 디버그 및 진단 데이터 |

## 라이브러리 컨텍스트 vs 리더 컨텍스트

ACP Chat은 두 가지 컨텍스트 모드를 지원하며, 플러그인이 현재 컨텍스트 유형을 자동으로 감지하여 백엔드로 전달합니다:

| 모드 | 설명 | 사용 사례 |
|------|-------------|-----------| 
| **라이브러리 컨텍스트** | Zotero 항목 목록에서 현재 선택된 항목을 기반으로 함 | 라이브러리를 탐색하는 중 빠른 참조 |
| **리더 컨텍스트** | Zotero 리더에서 현재 열려 있는 논문의 전문을 기반으로 함 | 심층 독서 중 맥락적 이해가 필요한 경우 |

## 세션 관리

- 대화 이력은 자동으로 영구 저장됩니다
- 백엔드별 다중 세션이 독립적으로 관리됩니다
- 대시보드 또는 사이드바에서 과거 세션을 확인할 수 있습니다
- 백엔드별 그룹화 세션 목록을 지원합니다

## 참고 사항

- 먼저 [ACP 백엔드](../backends/acp)를 설정해야 합니다
- 서로 다른 ACP 백엔드의 대화는 서로 간섭하지 않습니다
- 대화는 Zotero 항목과 연결되어 나중에 쉽게 참조할 수 있습니다
