# 사이드바 개요

## 사이드바란?

사이드바는 Zotero Agents가 제공하는 편리한 운영 패널로, Zotero 메인 창 우측에 플로팅됩니다. 현재 작업 컨텍스트를 벗어나지 않고도 백엔드와 상호작용하고, 실행 상태를 확인하고, 스킬 실행을 관리할 수 있습니다.

## 열기 방법

- **툴바 버튼**: Zotero 툴바의 사이드바 토글 버튼을 클릭합니다
- **메뉴**: **Tools → Open Sidebar**
- **대시보드 액션**: 대시보드에서 "Open/Close Sidebar"를 클릭합니다

## 아키텍처 설명

사이드바는 **iframe 아키텍처**를 사용합니다: 세 개의 탭이 각각 독립적인 HTML 페이지를 자식 iframe으로 로드하며, postMessage를 통해 플러그인 메인 프로세스와 통신합니다. 이 설계는 탭이 서로 간섭하지 않도록 보장하며, 각 패널은 독립적인 렌더링 컨텍스트를 가집니다.

워크스페이스 모드에서는 세 개의 탭이 통합 컨테이너에 통합되며, 레거시 모드에서는 각 패널을 Zotero의 라이브러리 창과 리더 창에 직접 임베드할 수도 있습니다.

## 세 개의 탭

| 탭 | 기능 | 사용 사례 |
|-----|----------|-----------| 
| **ACP Chat** | 현재 항목을 컨텍스트로 ACP 백엔드와 대화 | 문헌 열독 중 질문, 작성 지원 |
| **ACP Skills** | ACP 백엔드를 통해 실행된 스킬 실행을 모니터링하고 관리 | 실행 진행 상황 확인, 결과 검토, 권한 요청 처리 |
| **SkillRunner** | Skill-Runner 백엔드 실행을 확인하고 상호작용 | 인터랙티브 실행 관리, 인증 처리 |

## 인터페이스 가이드

### 탭 전환

사이드바 상단의 탭 바에서 세 개의 패널 간 전환이 가능합니다. 전환 시 이전 탭의 상태가 유지됩니다.

### 너비 조정

사이드바의 너비는 왼쪽 테두리를 드래그하여 자유롭게 조절할 수 있으며, 다양한 콘텐츠 표시 요구에 대응할 수 있습니다.

### 공통 컴포넌트

모든 탭은 다음 공통 UI 컴포넌트를 공유합니다:

- **배너**: 현재 선택된 프로젝트 정보와 액션 버튼을 표시하는 상단 정보 바
- **트랜스크립트 뷰**: 대화 또는 실행 로그를 위한 메인 영역으로, Plain 및 Bubble 표시 모드를 지원
- **응답 영역**: 메시지나 회신을 보내기 위한 하단 입력 영역
- **서랍 패널**: 좌우 양쪽에 확장 가능한 상세 정보 패널
- **프롬프트 컴포넌트**: 사용자 상호작용이 필요할 때 표시되는 프롬프트
- **플랜 컴포넌트**: 다단계 계획의 시각적 진행 상황
