# Literature Search & Ingest

## 目的

AI経由で学術文献を検索し、結果をZoteroに直接取り込みます。複数の検索モードに対応し、取り込み操作の実行前にインタラクティブな確認を行います。

## ユースケース

- 新しいトピックを研究する際に関連文献を検索・バッチ取り込みする
- 既知の論文のタイトル、DOI、arXiv ID、PMIDを入力してクイックインポートする
- シード論文に基づいて関連文献の検索を拡張する

## 入力制約

| 制約タイプ | 説明 |
|---------|------|
| 入力ユニット | workflow（アイテムを選択する必要はありません） |
| 実行方法 | コンテキストメニューまたはダッシュボードから実行、アイテムの事前選択不要 |

## 検索モード

| モード | 説明 |
|------|------|
| `auto` | 最も適切な検索モードを自動判定（デフォルト） |
| `topic_expansion` | 研究方向やトピックで検索し、関連文献を見つける |
| `paper_seed_expansion` | シード論文に基づいて検索を拡張 |
| `targeted_ingest` | 単一の論文を正確に特定して取り込む |

## 実行フロー

```
1. プラン確認フェーズ
   └── ZoteroライブラリとSynthesisコンテキストを読み取る
       └── 検索モードを自動判定（autoモード）
       └── 検索プランをユーザーに提示
       └── ユーザーの確認を待つ

2. 検索フェーズ（取り込みなし）
   └── 確認済みプランに従って候補文献を検索
       └── 検索結果リストを表示
       └── ユーザーが取り込む文献を選択

3. 取り込みフェーズ
   └── zotero-bridge経由で論文を1件ずつ取り込み
       └── メタデータインポートとPDF添付ファイルのインポートを含む
       └── 取り込み進捗を表示

4. 完了
   └── 取り込み結果サマリーを出力
       └── 成功/失敗したアイテムの情報を含む
```

### 操作の詳細

- このWorkflowは**インタラクティブ**モードで実行され、重要なポイントでユーザーの確認を必要とします
- プラン確認: AIが検索プランを提示した後、ユーザーが確認または調整します
- リスト確認: 検索結果が表示された後、ユーザーが取り込むアイテムをチェックします
- 実行進捗はダッシュボードで監視できます

## モデル推奨

🔴 ウェブ検索機能が必要です。このWorkflowの核心はオンラインで学術文献を検索することです — ウェブ検索機能のないモデルではこのタスクを実行できません。
🟢 モデルの推論能力は高くなくても構いません — 検索と取り込みは本質的に検索とツール呼び出しのタスクであり、軽量モデルでも処理できます。

## 出力

- 検索結果はZoteroアイテムとして直接取り込まれます
- PDF添付ファイルのダウンロードを自動的に試行します（ベストエフォート）
- 目標Collectionを指定して分類できます

## パラメータ

| パラメータ | タイプ | 説明 | デフォルト |
|------|------|------|--------|
| `query` | string | 検索トピック、研究方向、論文タイトル、DOI、arXiv ID、PMIDなど | — |
| `searchMode` | string | 検索モード | `auto` |
| `targetCollection` | string | 対象Collection（任意） | 空 |

### searchModeの利用可能な値

- `auto`: 自動判定
- `topic_expansion`: トピック拡張
- `paper_seed_expansion`: シード論文拡張
- `targeted_ingest`: 対象取り込み

## 依存関係

- **バックエンド**: ACPバックエンド（ACPプロトコルサポートが必要）
- **スキル**: `literature-search-ingest`スキルがバックエンド上にデプロイされている必要がある
