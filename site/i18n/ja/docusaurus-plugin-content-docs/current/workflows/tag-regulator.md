# Tag Regulator

## 目的

統制語彙に基づいてZoteroアイテムのタグを正規化し、AIを使用して可能な新しいタグを推論します。

このWorkflowは、Skill-Runnerバックエンド上で`tag-regulator`スキルを呼び出し、タグが語彙に準拠しているかをチェックし、関連するタグを推奨します。

## ユースケース

- 非標準タグをバッチでクリーニングする
- 既存の統制語彙に基づいてアイテムにタグを自動推奨する
- 統制語彙の継続的な更新と精緻化を維持する

## 入力制約

| 制約タイプ | 説明 |
|---------|------|
| 入力ユニット | 親アイテム |
| データソース | 親アイテムから取得: 現在のタグ、メタデータ（タイトル、著者、要約など） |

literature-analysisによって生成されたダイジェストMarkdown埋め込みペイロードが存在する場合、Workflowはそれをオプションのコンテキストとして自動的にアップロードし、推論品質を向上させます。

### 実行方法

- 1つ以上のZoteroアイテム（親アイテム）を直接選択
- アイテム選択後、コンテキストメニューから「Tag Regulator」を選択

## 実行フロー

```
1. 統制語彙の読み込み
   └── Zotero設定からtagVocabularyJsonを読み取る
       └── 語彙内の有効なタグリストを解析

2. リクエスト構築
   └── 親アイテムのメタデータと現在のタグリストを収集
       └── 統制語彙を一時的なYAMLファイルに書き出し
       └── Skill-Runnerにアップロード

3. Skill-Runner処理
   └── skill_id: "tag-regulator"を呼び出し
       └── タグの準拠性をチェック
       └── 推奨タグ（suggest_tags）を生成

4. 結果返却
   └── タグ変更を適用（非準拠タグの削除、推奨タグの追加）
       └── 推奨タグを現在のローカル語彙と照合
       └── 推奨タグを処理（ポップアップ操作）
```

### タグ処理ロジック

- **remove_tags**: 統制語彙にない現在のタグは削除されます
- **add_tags**: メタデータから推論されたタグがアイテムに直接追加されます
- **suggest_tags**: AIが提案した新しいタグで、ユーザーの確認が必要です
- **digest_markdown**: オプションの充実コンテキスト。ダイジェストMarkdown埋め込みペイロードが存在する場合のみアップロードされます

### リアルタイム同期ルール

結果が返されると、最新のローカル状態が読み込まれます:

- `suggest_tag`がすでに統制語彙に入っている場合、ポップアップは表示されません。`add_tags`のセマンティクスでアイテム更新に参加します
- `suggest_tag`がすでにステージングエリアにある場合、ステージングエリアに再書き込みされません
- 未処理のままの提案のみがポップアップに入ります

### 推定所要時間

| シナリオ | 論文あたりの推定時間 |
|------|-------------|
| ダイジェストなし（Literature Analysis未実行） | 約1分 |
| ダイジェストあり（Literature Analysis実行済み） | 1〜3分 |

アイテムにすでにダイジェストがある場合、AIはサマリーを追加コンテキストとして使用するため、推論はより精密になりますが時間がかかります。

### 推奨タグポップアップ

`suggest_tags`に対して、ダイアログで処理方法を選択できます:

- **追加**: 統制語彙に直接追加
- **ステージ**: ステージングエリアに配置し、後でレビュー
- **拒否**: 提案を無視
- **すべて追加 / すべてステージ / すべて拒否**: バッチ処理

ダイアログには10秒の自動ステージカウントダウンがあり、タイムアウトすると提案は自動的にステージングされます。

## 出力

### 1. タグ変更
- **remove_tags**: アイテムから語彙にないタグを削除
- **add_tags**: アイテムに推奨タグを追加
- 選択されたZoteroアイテムに直接適用

### 2. 推奨タグ処理
- ポップアップで処理方法を選択
- 承認されたタグ: `tagVocabularyJson`設定に追加
- ステージされたタグ: `tagVocabularyStagedJson`設定に追加

## モデル推奨

🟢 軽量モデルで十分です — タグ正規化は本質的に単純な分類・マッチングタスクであり、最強のモデルは必要ありません。

## パラメータ

| パラメータ | タイプ | 説明 | デフォルト |
|------|------|------|--------|
| `infer_tag` | boolean | タグ推論を有効にするかどうか | `true` |
| `valid_tags_format` | string | 語彙フォーマット | `yaml` |
| `tag_note_language` | string | 提案説明の言語 | `zh-CN` |

### valid_tags_formatの利用可能な値

- `yaml`: YAMLフォーマットを使用
- `json`: JSONフォーマットを使用
- `auto`: 自動検出

## 依存関係

- **統制語彙**: あらかじめ統制語彙を作成する必要があります。[タグ管理](../synthesis/tags)を参照
- **バックエンド**: Skill-Runnerサービス
- **バックエンド設定**: バックエンドマネージャーでSkill-Runnerタイプのバックエンドを設定
- **スキル**: `tag-regulator`スキルがSkill-Runner上にデプロイされている必要がある
