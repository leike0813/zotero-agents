# SkillRunnerタブ

SkillRunnerタブは、Skill-Runnerバックエンド経由で実行された実行を表示・操作するために使用されます。一回限りのスキル実行に焦点を当てるACP Skillsとは異なり、SkillRunnerタブはインタラクティブセッションの管理を重視しています。

## インターフェース概要

![SkillRunnerパネル](/img/docs/sidebar/skillrunner-tab.png)

```
┌─────────────────────────────────────┐
│  バナー: タイトル / requestId / ステータス    │
├─────────────────────────────────────┤
│  ← タスクドロワー  │  メインコンテンツエリア   │  詳細 → │
│               │  トランスクリプトビュー          │
│  実行中        │  プランコンポーネント            │
│  └─ backend1  │  プロンプトコンポーネント          │
│     └─ タスク A │  返信エリア                  │
│  完了          │                             │
│  └─ backend1  │                             │
│     └─ タスク B │                             │
└─────────────────────────────────────┘
```

## バナー

バナーには、現在選択されているタスクの情報が表示されます:

- **タイトル**: タスク名またはスキル識別子
- **リクエストID**: タスクの一意のリクエスト識別子
- **ステータス**: 実行ステータス（実行中 / waiting_user / waiting_auth / 完了 / 失敗）
- **バックエンド**: バックエンド情報
- **エンジン**: 使用中のエンジン（例: gemini、claudeなど）
- **モデル**: 使用中のモデル
- **更新時刻**: 最終更新時刻
- **タスクキャンセルボタン**

## タスクドロワー（左）

左側のドロワーには、すべてのSkillRunnerタスクが表示され、実行中と完了のグループに分割されています。各タスクエントリにはサマリー情報、ステータスインジケーター、アーカイブアクションが表示されます。エントリをクリックすると、そのタスクの詳細ビューに切り替わります。

## メインコンテンツエリア

### トランスクリプトビュー

SkillRunnerのトランスクリプトビューは、継続的な推論をインテリジェントに処理する**Thinkingチャットモデル**を使用します:

- **思考ブロック**: AIの推論プロセスは、独立した思考ブロックとして表示
- **ツール呼び出し**: ツール名、入力サマリー、実行ステータスを表示
- **メッセージ**: アシスタントとユーザーの対話メッセージ
- **リビジョン**: 出力バージョンの変更記録

**Plain / Bubble**表示モードにも対応しています。

### 認証ワークフロー

SkillRunnerタブは認証ワークフローに対応しており、パネルを離れることなくバックエンドの認証を完了できます:

**認証トリガー:**

- 認証を必要とするスキルを実行する際に自動的にトリガー
- プロンプトコンポーネントに認証リクエストが表示

**対応する認証方式:**

| 方式 | 説明 | ユースケース |
|------|------|---------|
| **OAuthプロキシ** | ブラウザを介してOAuthフローを完了 | 推奨方式。OAuthをサポートするエンジン向け |
| **認証コード入力** | 認証コードまたはURLを手動で入力 | エンジンが認証リンクを生成した場合 |
| **ファイルインポート** | 認証情報ファイルをインポート | 認証情報ファイルが既に利用可能な場合 |
| **インラインTUI** | パネル内で直接ターミナルを起動 | インタラクティブなログインが必要な場合 |

**認証フロー例（OAuth）:**

1. 実行で認証が必要であることを検出
2. プロンプトコンポーネントに「認証が必要」と利用可能な認証方式が表示
3. ユーザーがOAuthプロキシを選択
4. ブラウザでOAuthページが開く
5. ユーザーが認証を完了
6. 実行が自動的に再開

### プロンプトコンポーネント

| ステータス | 表示内容 |
|------|---------|
| `waiting_user` | ユーザー入力を待機中。コンテキスト説明とクイックオプション（利用可能な場合）を表示 |
| `waiting_auth` | 認証を待機中。認証方式の選択と入力を表示 |
| `running` | 実行中のインジケーター |
| `completed` | 完了ステータスの確認 |
| `error` | エラー情報とトラブルシューティングの提案 |

### 返信エリア

- **テキスト入力ボックス**: 返信内容を入力
- **送信/キャンセルボタン**

ACP Skillsとは異なり、SkillRunnerタブの返信エリアにはモード/モデル/推論セレクターがありません（これらはバックエンド設定で構成されます）。

## 詳細ドロワー（右）

| エリア | 内容 |
|------|------|
| **実行メタデータ** | タイトル、requestId、taskKey、ステータス、終了/待機フラグ |
| **バックエンド情報** | バックエンド、エンジン、モデル |
| **更新時刻** | 最終アクティブ時刻 |
| **操作情報** | 現在の保留中の操作情報（ある場合） |
| **セッションサマリー** | 履歴セッションのサマリー |
| **リビジョンサマリー** | 出力バージョンの変更記録 |

## 関連設定

SkillRunnerタブを使用する前に、[Skill-Runnerバックエンド](../backends/skill-runner)を設定する必要があります。
