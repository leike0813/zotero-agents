# トピック統合の作成

## トピック統合とは

トピック統合（Topic Synthesis）は、関連する文献群を体系的に分析・統合するプロセスである。AI Workflow を通じて重要情報を自動抽出し、トピック構造を特定し、包括的な分析レポートを生成する。

## Topics サーフェス

Synthesis Workbench → Topics ページでは、作成されたすべてのトピックを閲覧・管理できる。Topics サーフェスは**3 つのビューモード**をサポートする。

| ビュー | 説明 | ユースケース |
|--------|------|-------------|
| **グラフビュー** | トピックをノード、関係をエッジとする力指向グラフ | トピック間の関連性を直感的に把握 |
| **グリッドビュー** | タイトル、論文数、サマリー、アクションボタンを含むカード | トピックの閲覧と検索 |
| **リストビュー** | テーブルビュー（列：名前、論文数、作成時刻、更新日、ステータス） | 並び替えと一括操作 |

![Synthesis Topics グラフビュー](/img/docs/synthesis/topic-graph.png)

### トピック管理操作

- **検索**: トピック名と説明で検索
- **並び替え**: タイトル、論文数、更新日で並び替え
- **新規トピック作成**: 作成ボタンをクリックして Workflow パイプラインを開始
- **トピック更新**: パイプラインを再実行してトピック分析を更新
- **トピック削除**: 不要になったトピックを削除

## 作成プロセス

トピック作成は Workflow によって駆動され、多段階の自動化パイプラインである。

```
1. create-topic-prepare
   → 文献データを収集し、論文セットを構築
   
2. topic-synthesis-core-enrichment
   → コアエンリッチメント：情報を抽出し、知識を関連付け
   
3. topic-synthesis-finalize
   → 最終分析アーティファクトとレポートを生成

（update-topic-synthesis-prepare は既存トピックの更新に使用）
```

### 前提条件

- [Skill-Runner バックエンド](../backends/skill-runner)が設定済みであること
- ライブラリに関連する論文があること
- 論文にダイジェストと引用分析が生成されていること（オプション、推奨）

このパイプラインは [Topic Synthesis Creation](../workflows/topic-synthesis) Workflow によってオーケストレーションされる。

## トピックインスペクター

トピック作成後、クリックするとトピックインスペクターに入る。これは 8 つのサブページを含むマルチページリーダーで、各ページがトピックの異なる側面を提示する。

### Overview

- トピック名、説明、重要度スコア
- コア主張のサマリー
- 統計情報（論文数、カテゴリ数、主張数など）
- トピックグラフでの位置情報

### Taxonomy

トピックの階層分類構造を表示する。

- より広範なトピック：上位のトピック領域
- より狭いトピック：より具体的なサブトピック
- 関連トピック：その他の関連トピック
- トピックグラフにおける位置と階層

### Claims

文献から抽出されたコア主張とアサーション。

- 各主張には元の証拠引用を含む
- 主張の由来となる論文をマーク
- 主張タイプ（発見/仮説/結論など）
- 主張を支持する論文数

### Compare

同じトピックに関する異なる論文間の視点比較。

- 比較の次元（方法、結論、データセットなど）
- 各論文の立場と議論
- 合意と分岐の可視化

### Future Directions

文献分析を通じて特定された研究ギャップと将来の方向性。

- 未解決の疑問
- 潜在的な研究方向
- 関連する課題と提言

### Coverage

トピックが関連文献をどの程度カバーしているかを分析する。

- トピックがカバーする論文のリスト
- 論文の完全性（ダイジェスト/引用分析アーティファクトの存在有無）
- カバーされている側面とカバーされていない側面

### References

トピックに関連付けられたすべての参考文献。バインディングの詳細を含む。

- 各引用の Zotero アイテムリンク
- トピックにおける引用の役割（support / contrast / background）
- 引用のソースとコンテキスト

### Report（完全レポート）

生成された構造化統合分析レポート（Markdown 形式）。

- 完全なトピック分析テキスト
- Markdown または自己完結型 HTML としてエクスポート可能
- 学術執筆の参考文献として使用するのに適している

## トピックグラフ

トピックグラフは、トピック間の関係を示す階層的なトピックネットワークである。

### ノードタイプ

| タイプ | 説明 |
|--------|------|
| **materialized** | 実際に作成された構造化トピック |
| **placeholder** | 存在すると推論されたが、まだ作成されていないトピックプレースホルダー |

### エッジステータス

| ステータス | 説明 |
|-----------|------|
| `suggested` | システムが提案した関係（レビュー待ち） |
| `confirmed` | ユーザーが確認した関係 |
| `rejected` | ユーザーが却下した関係 |
| `stale` | 古いデータ。再評価待ち |
| `deleted` | 削除された関係 |

### 関係タイプ

| 関係 | 説明 |
|------|------|
| `broader_than` | A は B よりも広範なトピックである |
| `related_to` | 2 つのトピックが関連している |
| `overlaps_with` | 2 つのトピックが重複している |
| `contrasts_with` | 2 つのトピックが互いに対照的である |

### トピックの管理

- **新規トピック作成**: Topics ページで「作成」をクリック
- **トピック編集**: 名前、説明、重要度などを修正
- **論文の関連付け**: トピックから論文を追加または削除
- **トピックグラフの閲覧**: トピック間の関係ネットワークを表示
